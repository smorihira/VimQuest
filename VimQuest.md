## 1. プロダクトビジョン

### 1.1 コンセプト
VimQuestは、Vimの操作を「ライフ（体力）を管理しながら解く戦略パズル」に変えるWebアプリ。既存の学習ツールが「操作をなぞるだけ」なのに対し、本作は「限られたリソースでいかにスマートに目的の状態へテキストを変換するか」という知略を問うゲーム体験を提供する。

### 1.2 ターゲット
「Vimのポテンシャルには惹かれるが、結局VSCodeの方が楽」と感じて挫折したVim初〜中級者。`h j k l`や`Backspace`の連打から抜け出せない層。

### 1.3 提供価値
Vimの学習を「リソース管理パズル」へ昇華させ、能動的に効率的な手順を探す楽しさを提供する。「黒い画面で難しそう」というテッキーなイメージを払拭し、親しみやすいパズルゲームとして実務で使えるVim操作を自然に身につけさせる。

### 1.4 既存サービスとの差別化

| サービス | フォーカス | VimQuestの独自性 |
|---|---|---|
| Vim Adventures | 指の慣れ（RPG風） | 戦略的な編集（パズル風） |
| VimBeGood | 反射神経・速度 | 思考の質・ライフ管理 |
| VimGolf | 競技・最短手数 | 教育的な制限・ステップアップ設計 |

## 2. 核心的なゲームシステム（Core Mechanics）

### 2.1 ライフ（体力）制と評価システム

- **基本ルール**: 1コマンド実行ごとにライフを1消費する（＝1ダメージ）。`ciw`や`dw`などの複合コマンドも、`x`や`j`などの単キーも、すべて**1コマンド＝1ダメージ＝ライフ1消費**。同じ結果を達成するのに`x`連打は5ダメージ、`dw`なら1ダメージという差が自然に生まれ、Vimの「動詞＋対象」合成の価値を体感させる。

#### ダメージルール詳細

| 操作 | ダメージ |
|---|---|
| ノーマルモードのコマンド（`dw`, `ciw`, `o`, `x`等） | 1 |
| インサートモード中のテキスト入力（最初の5文字） | 0（無料） |
| インサートモード中のテキスト入力（6文字目以降） | 5文字ごとに+1 |
| `Esc`（インサート/Visual→ノーマル復帰） | 0 |
| モード即キャンセル（`i`→即`Esc`、`v`→即`Esc`等） | 進入コストも取消し（0に戻る） |
| `/検索語` + `Enter` | 1 |
| `n` / `N`（検索結果移動） | 1 |
| 数値プレフィクス付きコマンド（`3w`, `5j`, `2dd`等） | 1（カウントに関係なく1コマンド扱い） |
| `u`（アンドゥ） | ダメージも巻き戻る |

> 入力コストの設計意図: `ciw`で変数名を置換（3〜5文字）は追加コスト0。行全体の打ち直し（30文字）は+5ダメージ。「ピンポイント編集は安い、雑な打ち直しは高い」というVimの価値観をコストで体現する。
- **ライフ設計（星評価ベース）**: 各ステージに対して「☆3解法」「☆2解法」「☆1解法」を先に設計し、それに基づいてライフ総量と星の閾値を決定する。例: ☆3解法＝3手、☆2解法＝5手、☆1解法＝8手 → ライフ＝10（☆1解法＋余裕2）。「☆3を狙う再挑戦」のモチベーションを自然に生む。
- **Undoによる回復**: `u`を実行すると、テキストの状態と共に消費したライフも一手前へ回復。無料・無制限。初期状態まで全て戻せる。`Ctrl+R`でredoも可能。死を恐れない試行錯誤を保証する。
  - **undo時UI**: テキスト巻き戻り+変更箇所success色フラッシュ、ライフゲージ回復アニメ、合成カード分解（合成の逆再生、0.2秒）、ステータスラインに`u`表示+巻き戻しコマンド取消線
  - **SPELL記録**: undoされた操作は消える。redo（`Ctrl+R`）で操作がSPELLに復活する。クリア時点の操作列が最終記録となる（「磨き上げた解法」を称える）

#### 星評価（三段階）
ステージクリア時の残りライフ数に応じて評価を算出。
- ☆3: 理想的な「Vimらしい」解法に近い
- ☆2: 若干の冗長な操作が含まれるが、標準的な効率
- ☆1: ギリギリのライフでクリア

### 2.2 ステージクリア条件

- **基本ゴール**: 「初期テキスト」を操作して「ゴールテキスト」と完全一致させる（テキスト一致型）。差分が0になった瞬間クリア。
- **拡張条件（後半ステージ）**: テキスト一致に加え、カーソル位置やレジスタの状態もクリア条件に含む。例:「指定テキストをヤンクした状態で終了せよ」→ 次ステージでペーストを使う伏線とし、「コピー→移動→貼り付け」ワークフローを自然に教える。
- **インサートモード**: プレイヤーが実際にテキストを入力する。インサートに入るコマンド（`ciw`, `i`, `o`等）で1ダメージ。入力テキストは最初の5文字まで無料、6文字目以降は5文字ごとに+1ダメージ（詳細はダメージルール表参照）。Enterも通常の文字入力として文字数カウントに含む（改行=1文字）。Vimの実際の操作フロー（Normal→Insert→Normal）を正しく体験させる。
- **検索（`/`）**: プレイヤーが検索語を実際に入力する。`/検索語` + `Enter`で確定、`n`/`N`で次/前のマッチに移動。スキルツリー後半で解禁、長文ファイルのステージと組み合わせて導入。

### 2.3 スキルカード（手札）システム

- **手札＝使用可能コマンド**: 各ステージで使えるコマンドは「手札」として制限される。スキルツリーで習得済みのコマンドのみが手札に入り、未習得のコマンドは使用不可。未習得キーを押した場合はダメージなしでエラーフィードバック（視覚的なフラッシュ等）を返し、「まだ使えない」ことを伝える。
- **Composable UI**: 「動詞（`c`, `d`）」と「対象（`iw`, `t`, `f`）」が組み合わさる際、UI上で合体し強い光を放つ演出を行い、「意味の合成」を視覚化する。
- **基本コマンドの昇格**: チュートリアルを終えた基本コマンド（`h j k l`, `x`, `i`/`a`, `Esc`, `w b e`, `0 $`等）はカードとして表示せず、常時使用可能な「習得済みスキル」に昇格する。カードパネルには合成系・テキストオブジェクト系など戦略的なコマンドのみを表示し、UIのノイズを防ぐ。

### 2.4 ステージ進行構造（スキルツリー型）

Vimのコマンド体系（動詞×対象の組み合わせ）に基づくツリー状の進行マップ。基本移動→単純な編集→動詞と移動の合成→テキストオブジェクトと進み、コマンドの依存関係が可視化される。「暗記ではなく組み合わせ」であることを構造的に伝え、ツリーの先に進むほど手札が増え、解法の幅が広がる。

#### 学習順序（移動と編集の交互型）
移動と編集を交互に導入し、「覚えたコマンドがすぐ役立つ」サイクルを維持する（Teach → Test → Rewardループ）。移動だけ・編集だけが続く区間を避け、離脱を防ぐ。

#### 初期ツリー構造
```
[h j k l] ─→ [x] ─→ [w b e] ─┬→ [dw / de / db]
                                └→ [0 $] ─→ [d$ / d0]
```
1. `h j k l` — 基本移動。最小限の移動でカーソルを目標位置へ
2. `x` — 初めて「テキストを変える」体験。ただしすぐライフが厳しくなる
3. `w b e` — 単語移動。移動の効率が劇的に上がる「最初の感動」
4. `dw / de / db` — 動詞`d`と移動の初めての合成。スキルカード合体演出の初登場
5. `0 $` — 行頭/行末。行内の長距離移動
6. `d$ / d0` — 合成の応用

ステップ2→4で「`x`連打がつらい→`dw`一発で解決！」という渇望→報酬サイクルを早期に体験させる。

#### スキルツリー全体図
```
[h j k l] ─→ [x] ─→ [w b e] ─┬→ [dw / de / db] ─→ [dd] ─→ [d$ / d0]
                │               │                    │    └→ [D]
                │               ├→ [0 $] ─→ [^] ─────┘
                │               │
                └→ [W B E]      └→ [f / t] ─┬→ [; / ,]
                                             ├→ [%]
                                             ├→ [df / dt / cf / ct]
                                             │
                                             └→ [テキストオブジェクト: iw/aw, i"/a", i'/a', i(/a(, i{/a{, i[/a[, i</a<, it/at]
                                                  │
                                                  ├→ [ciw / caw / ci" / ca( 等] ─→ [s / S / C] ─→ [.（ドットリピート）]
                                                  │
                                                  ├→ [diw / daw / di" / da( 等]
                                                  │
                                                  ├→ [gu / gU]（guiw, gUiw 等）
                                                  │
                                                  ├→ [>> / <<]（Visual中の > / < も含む）
                                                  │
                                                  ├→ [v / V] ─→ [Ctrl+v（矩形選択）]
                                                  │
                                                  └→ [y系: yw, yy, yiw] ─→ [p / P]
                                                       │
                                                       └→ [レジスタ: "a〜"z, "0]

[i / a] ─→ [I / A] ─→ [o / O]

[r]（独立ノード。x習得後に分岐）
[~]（独立ノード。r と並列）
[J]（独立ノード。dd 付近で解禁）

[{count}]（独立ノード。中盤以降で解禁、既存コマンドすべてに適用可能）

[gg / G / {数字}G] ─→ [Ctrl+d / Ctrl+u] ─→ [zz / zt / zb]
                                             ─→ [/ 検索 + n/N] ─→ [* / #]

[gj / gk]（独立ノード。長文ステージ導入時に解禁）
```
- テキストオブジェクトが中心のハブ。`d`/`c`/`y`/`gu`/`gU`すべてと組み合わさる
- `f/t`をテキストオブジェクトの前提に配置（行内の「特定文字まで」の概念を理解してからオブジェクトへ）
- `%`は`f/t`と並列（括弧ジャンプも「特定位置への移動」の一種）
- `^`は`0 $`の後（インデントされたコードを扱い始めるタイミング）
- `W B E`は`w b e`の後に独立分岐（大きな単位の移動）
- `s`/`S`/`C`は`c`系の後にショートカットとして解禁（`cl`→`s`、`cc`→`S`、`c$`→`C`の短縮形であることを体験させる）
- `.`は`s`/`S`の後に解禁（`ciw`→`.`の流れが最も感動的）
- `gu`/`gU`と`>>`/`<<`はテキストオブジェクトと組み合わせる動詞として配置
- Visualモードはテキストオブジェクト習得後に解禁（`viw`/`vi"`を最初から使えるようにし、非効率な`v`→移動連打を防ぐ）
- `*`/`#`は`/`検索の後（検索を理解した上でのショートカット）
- `zz`/`zt`/`zb`は`Ctrl+d`/`Ctrl+u`の後（スクロール概念の拡張）
- `J`/`r`/`~`は独立ノード。他のコマンドと依存関係が薄い単体操作
- `gj`/`gk`は長文ステージ導入時に解禁

#### ドットリピート（`.`）の設計
`.`はスキルツリー中盤以降で解禁。解禁前に「`ciw`→`w`→`ciw`→`w`→`ciw`」と同じ操作を手動で繰り返す苦痛を体験させた上で`.`を与え、「渇望→報酬」サイクルの最高峰の感動を演出する。`.`自体のコストは通常通り1コマンド＝1消費（それでも十分お得）。

**`.`のカード演出**: `.`カードは手札に表示、ホバーで「前回: ciw」のように直前コマンドを表示。押下時、前回のカード合成演出をゴースト（半透明）で0.15秒高速再生し、「何が繰り返されたか」を視覚的に伝える。

## 3. UI/UXデザイン

### 3.1 ミニマル・モダンアートデザイン
初心者の認知負荷を下げるため、無駄を削ぎ落とした「ミニマル・モダンアート風」のクリーンなデザイン。

- **モードの視覚化**: 画面全体の背景色で現在のモードを伝える（Normal＝青、Insert＝オレンジ、Visual＝紫 `#2a1a3e`）。文字を読まなくても状態を直感的に把握可能。
- **カーソル表示**: Normal=ブロックカーソル（█）、Insert=ラインカーソル（｜）、Visual=ブロック+選択範囲ハイライト。実Vimと同じ形状。カーソル点滅はなし、代わりに微かなグロー（発光）で位置を強調。Visualの選択範囲は `rgba(124,124,255,0.25)` （accent半透明）。
- **ナビゲーター**: 画面隅の幾何学的な浮遊キューブ。以下の2つの役割を担う：
  - **リアクション役**: v1.0最小セット — 常時ゆっくり回転、効率的な操作で一瞬発光、非効率な操作で一瞬暗く、ライフ低下で回転が遅くなり暗くなる。細かいリアクション（undo逆回転、クリア激しく輝く等）はv1.1以降。プレイテストで調整
  - **ヒント役**: 詳細はセクション4.2を参照
- **聴覚フィードバック**: コマンド実行時に心地よい打鍵音を再生（寿司打のような気持ちよさ）。v1.0ではまず1種類の汎用打鍵音で統一し、効率/非効率の音の区別はv1.1以降で追加検討。不正コマンド入力時は「不発音」（別の短い音）で無効であることを伝える。
- **シンタックスハイライト**: エディタ内のコードは正規表現ベースの軽量トークナイザでフルハイライト。v1.0対象言語: CSS, HTML, JavaScript, JSON, Markdown, Python。配色: キーワード=accent紫(`#7c7cff`)、文字列=success緑(`#44ddaa`)、数値=star金(`#ffd700`)、コメント=グレー(`#666`)。「本物のエディタ」感を出し、コード構造の把握を助ける。

### 3.2 画面構成と遷移フロー

#### 画面一覧
| 画面 | ヘッダー | 説明 |
|------|---------|------|
| Landing | なし | 初回/URL直アクセス時のみ表示。1画面完結。ボタンは進捗がないとき「はじめる」、LocalStorageに進捗があれば「つづける」に自動切替 |
| Skill Tree | あり（ロゴ+進捗表示） | ホーム画面的位置付け |
| Play | なし | エディタの没入感優先 |
| Result | なし（オーバーレイ） | ステージクリア時 |
| Game Over | なし（オーバーレイ） | ライフ0時 |
| New Weapon | なし（オーバーレイ） | 新コマンドアンロック時 |

#### 画面遷移フロー
```
Landing ──→ Tree ←──────────────────┐
              │                      │
              ▼                      │
            Play ──→ Result ─────→ Tree
              │        │
              │        ├──→ Weapon ──→ Tree（ツリーで開放状態を確認させる）
              │        │
              │        └──→ Play（次ステージへ直接遷移OK）
              │
              └──→ GameOver ──→ Play（リトライ）
                      │
                      └──→ Tree
```
- Landingの再訪問導線は不要。Treeがホーム
- Result画面: 「もう一度」「次のステージへ」「ツリーへ」の3択。NEW WEAPON表示時は「次のステージへ」押下で内部的にWeapon画面を経由
- GameOver画面: 「リトライ」「ツリーに戻る」の2択
- Weapon画面: キー入力で自動遷移

#### ギブアップ導線
- プレイ画面の左上に控えめなアイコンで「ツリーに戻る」を配置。確認ダイアログなしの即時遷移（Vimの`:q!`=強制終了の意味を体現）
- `:q!` を入力するとツリーに戻れるイースターエッグ

#### プレイ中の画面レイアウト
- **上部**: テキストエディタ領域（メイン）。背景色でモード表示
- **下部**: スキルカード（手札）パネル。使えるコマンドをカード状に横並び表示。カードゲームの手札メタファーに沿い、常に視認可能。動詞と対象の合成時にカード同士が合体する演出あり
- **エディタ:カード比率**: 70:30の固定比率。コードが少ないステージでもカード領域が跳ね上がらないよう固定
- **カード表示**: コマンド名のみ大きく表示、ラベル（delete, change等）はホバーツールチップ。手札はv1.0最大12〜15枚想定。多い場合は2行表示（上段=動詞・モーション、下段=テキストオブジェクト・合成）
- **ステータスライン**: エディタ下部に1行のステータスライン。入力中のキーシーケンスを表示（`3`, `3d`, `3dw`等）。実Vimのフィードバックを学ぶ教育的役割
- **コマンド入力フィードバック**: ステータスライン表示とカード連動の両方。動詞カード（`d`等）押下時、有効な対象カードがアクティブ化、無効なカードはグレーアウト。カウント入力時はカード上に×Nバッジ表示
- **カード合成演出**: `d`+`iw`→`diw` の合体はインプレース変化。動詞カードがパルス→対象押下で動詞カードが合成名に変化+対象カードがフェードアウト→合成カードがエディタへ飛んで効果適用。「動詞が対象を食べて合成コマンドになる」概念の可視化。全体0.3〜0.5秒。合成完了時に打鍵音再生
- **不正コマンド入力**: 手札にないキーや無効なキー押下時、画面端が一瞬赤フラッシュ（vignette的）+ ステータスラインに(unknown)表示 + 不発音。ライフ消費なし・ダメージカウントなし
- **Insertモード時のカード領域**: Insert中はカードの代わりにInsert情報を表示。入力文字数カウント（`3/5 FREE` → `6/+1 DMG`）とEscリマインダー。「あと何文字無料か」を直感的に伝える
- **検索（`/`）のUI**: ステータスラインを流用。`/`入力中はステータスラインが検索入力欄に変化。マッチ箇所はstar色半透明（`rgba(255,215,0,0.3)`）でハイライト、現在マッチはより濃い色で強調
- **ステージ開始イントロ**: Play画面表示時、エディタ中央にステージ名＋目標テキストを1.5秒表示。任意キー/クリックでスキップ可能
- **左上隅**: ライフゲージ
- **右下隅**: ナビゲーター（浮遊キューブ）

### 3.3 ビジョン（透視）システム
- **Spaceキー切替表示**: Spaceキーをホールドしている間、現在テキストが消えゴールテキストのみが表示される。「完成図を頭に焼き付け、Spaceを離して記憶を頼りに操作する」というサイクルがゲーム性を生む。同時にこれは実務でのVim操作（変更箇所を確認→覚えて→操作する）のトレーニングにもなる。
- **切替演出**: Space押下で即座にゴールテキストに切替 + 変更箇所を0.2秒success色フラッシュで視線誘導。離すと即座に現在テキストに戻る（フラッシュなし）。「どこを変えればいいか」の気づきを促進。

### 3.4 リザルト演出

- **魔法の呪文表示（YOUR SPELL）**: クリア時、打ち込んだコマンド配列を美しくタイポグラフィ化。各コマンドの下に消費ダメージ数を小さく表示。ステージの学習対象コマンドは緑（success色）で強調、それ以外は通常accent色、Esc/テキスト入力はdim色。
- **スコア表示**: ラベルは `DAMAGE`（COMMANDS ではない）。消費ライフ総数を表示。LIFE（残り/最大）も併記。星の下にライフを重複表示しない（Stats欄に一本化）。
- **星表示**: 大きな星は今回の結果のみ。BESTは星ではなくテキスト（`BEST: 3 damage`）で表示。BEST更新時は `NEW BEST!` を金色で追加。
- **余韻のデザイン**: 他者との比較を捨て、自分の打鍵の美しさと向き合う。
- **演出順序**: シーケンシャル登場（全体2.5秒）。暗転中にSTAGE CLEAR(0.0s) → ステージ名(0.5s) → 星1つずつ点灯+効果音(0.8s) → YOUR SPELLスライドイン(1.5s) → Statsフェードイン(2.0s) → NEW WEAPONバウンス(2.2s) → ボタン群(2.5s)。クリック/キーで即全表示スキップ可能。

#### プレイ画面 Top bar
- ステージタイトルとモード表示のレイアウトは実装時に整理（ステージ番号+タイトル、モード表示、ライフを横並びで整理）

#### 画面遷移アニメーション
- 基本: 全遷移共通で暗転フェード（0.2秒）
- Play→Result: 暗転中にSTAGE CLEARテキストを0.5秒表示してからリザルト画面へフェードイン
- Play→GameOver: 画面が赤フラッシュしてからフェード
- その他: 統一フェードのみ

## 4. チュートリアルと救済システム

### 4.1 新武器習得
新しいキー（武器）が手に入る際、パーティクル演出と共に実際にキーを叩かせて「身体」に馴染ませるフェーズを設ける。TRY ITエリアでコマンドを正しく実行すると「次へ」ボタンが出現し、押下でTreeに遷移。成功必須で確実に1回は正しい操作を体験させる。

### 4.2 救済措置（ヒントシステム）
- **ナビゲータークリック**: ナビゲーター（浮遊キューブ）をクリックすると、初期状態からの想定解法（☆３解法）のオートプレイをオーバーレイで実演。見せるなら一気に全部見せる。ただしヒント使用時はそのステージは☆1確定。「次は自力で☆3を取りたい」という再挑戦動機を生む。同一プレイ中に何度でも再生可能（☆は初回使用で☆1確定済みのため追加ペナルティなし）。オーバーレイ表示のためモード（Normal/Insert/Visual）に関係なくいつでも再生でき、現在のエディタ状態はリセットされない。
- **ヒント使用時のResult表示**: ☆2・☆3をグレーアウト(`#444`)+🔒重ね表示。☆1のみ通常点灯、☆2・☆3でロック演出（一瞬光ろうとしてグレーに戻る+🔒、0.3秒）。☆下に「ヒント使用: ☆1確定」テキスト、横に「ヒントなしで挑戦」の情報テキスト。

### 4.3 ゲームオーバー時の体験
ライフが0になると、操作ログを表示し「どこでライフを浪費したか」をハイライト。連続する同一コマンドは畳んで表示（例: `j ×3 -3`、`x ×7 -7`）。waste操作は赤ハイライトで強調。ログ下部に合計ダメージサマリー（`合計ダメージ: 14 / ライフ: 10 → 4オーバー`）を表示し、HINTの最適解ダメージと対比して改善幅を可視化。HINTとして最適解（☆3解法の要点）を提示し、具体的な改善策に気づかせてからリトライへ導く。演出は短く、煩わしくならないこと。

## 5. ステージ構成とレベルデザイン

### 5.1 コンテンツ思想
最初は不便な操作を強いてライフを削らせ、その直後に解決策となる武器を与えて感動を生む「渇望」のレベルデザイン。

### 5.2 レベルデザイン原則
- **差分は小さく、外科的に**: 各ステージの初期テキストとゴールテキストの差分は小さく保つ。「全消し→全入力」よりもターゲット編集の方が常に安くなるよう設計し、エクスプロイトをレベルデザインで押さえる。これはVimの実務での使い方（ピンポイントな変更）とも一致する。
- **ステージ数は可変**: コマンドの複雑さと実務重要度に応じてステージ数を配分。重要なコマンド（`ciw`, `f`, `.`等）は複数ステージ、補助的なもの（`I/A`等）は1ステージ。数に縛られず設計する。

### 5.3 スキルツリー画面のUI
- 初期表示は全ノードが収まるようにズームレベルを自動調整
- マウスドラッグでパン、ホイールでズーム
- ミニマップはv1.0不要（60コマンド程度なら全体フィット可能）
- availableノード（次にプレイすべきステージ）にフォーカスするボタン `[🎯]` を右下フローティング丸ボタンで配置。押すとavailableノード群を中心にスムーズパン+ズーム（0.3秒）
- lockedノードのコマンド名は表示する（opacity低め）。「次にこれが手に入る」という渇望感の演出として重要。lockedノードクリック時は軽くシェイク（左右に2回揺れる）+ロック音
- ノードホバー: available/clearedノードはツールチップ（コマンド名・説明・★数・ステージ数）。lockedはコマンド名のみ
- レイアウトは左→右、ブランチは下に伸びる構造- **リプレイ仕様**: clearedノードクリックでリプレイ可能。同じ手札（そのステージ時点のカード）でプレイ。☆・BESTは更新される。ヒント使用フラグはリセット（ヒントなしで☆3に挑戦し直せる）
### 5.4 ステージデータに含む要素
- ステージID / ツリー上の位置
- ステージタイトル / フレーバーテキスト
- 初期テキスト / ゴールテキスト
- 初期カーソル位置
- 使用可能コマンド（手札）
- ライフ総量 / ☆3・☆2・☆1の閾値
- ☆3解法のコマンド列（オートプレイ / ヒント用）
- 拡張クリア条件（レジスタ状態、カーソル位置等。該当ステージのみ）
- 新武器習得フラグ（クリアで新コマンドをアンロック。ツリー分岐点のみ）

### 5.5 題材
日本語IMEトラブルを避けるため、すべて英数テキストに限定。
- **チュートリアル段階（新武器習得直後）**: 一般的な英語テキスト（単語リスト、簡単な英文など）。コマンドの使い方そのものに集中させる
- **実践段階（習得したコマンドを活用するステージ）**: Web開発コード中心（CSS、HTML属性、JSON設定、JavaScript関数など）。ターゲット層に馴染み深い題材でVimの実務的価値を体感させる

### 5.6 v1.0スコープ（単一ファイル完結）
v1.0は単一ファイル内の編集に完結させる。マクロ・複数ファイル操作はv2.0以降の拡張とする。スキルツリー型のため、将来のノード追加（新コマンドの分岐）が容易。

#### v1.0 コマンド完全リスト

**移動系**
- `h j k l` — 基本移動
- `w b e` — 単語移動
- `0 $` — 行頭/行末
- `^` — 行頭の非空白文字へ移動（インデントされたコードで`0`との違いが活きる）
- `W B E` — WORD移動（空白区切り。`w b e`より大きな単位で移動）
- `f{char}` / `t{char}` — 行内ジャンプ
- `;` / `,` — `f`/`t`の繰り返し（同方向/逆方向）
- `gg` / `G` — ファイル先頭/末尾
- `{数字}G` — 指定行ジャンプ

**編集系（動詞）**
- `x` — 1文字削除
- `d` + 移動/テキストオブジェクト — 削除（`dw`, `dd`, `d$`, `d0`, `df{char}`, `dt{char}`, `diw`, `di"`, `da(`等）
- `D` — カーソルから行末まで削除（`d$`のショートカット）
- `c` + 移動/テキストオブジェクト — 変更（`cw`, `cc`, `c$`, `cf{char}`, `ct{char}`, `ciw`, `ci"`, `ca(`等）
- `C` — カーソルから行末まで変更（`c$`のショートカット）
- `r` — 1文字置換
- `s` — 1文字削除してインサートモードへ（`cl`のショートカット）
- `S` — 行全体を削除してインサートモードへ（`cc`のショートカット）
- `y` + 移動/テキストオブジェクト — ヤンク（`yw`, `yy`, `yiw`等）
- `p` / `P` — ペースト

**テキストオブジェクト（`d`, `c`, `y`と組み合わせ）**
- `iw` / `aw` — 単語
- `i"` / `a"` — ダブルクォート内
- `i'` / `a'` — シングルクォート内
- `i(` / `a(` — 括弧内
- `i{` / `a{` — 波括弧内
- `i[` / `a[` — 角括弧内
- `i<` / `a<` — 山括弧内（HTMLタグ等）
- `it` / `at` — HTMLタグペアの中身（`<div>...</div>`）

**Visualモード**
- `v` — 文字単位ビジュアル選択（1ダメージ）
- `V` — 行単位ビジュアル選択（1ダメージ）
- `Ctrl+v` — 矩形選択（Visual Block、1ダメージ）。ブラウザの「貼り付け」を`preventDefault()`で奪取（ゲームエリアフォーカス時のみ）
- `I` / `A`（Visual Block中）— 矩形選択の各行先頭/末尾に同じテキストを一括挿入。`Ctrl+v`の真の威力。テキスト入力コストは実タイプ量ベース（3文字×5行でも3文字としてカウント）
- `o` — Visual中に選択範囲の反対端へカーソル移動（1ダメージ）。選択範囲の微調整に使用
- ビジュアル選択中の移動（各1ダメージ）、確定操作の`d`/`c`/`y`等（1ダメージ）もすべて通常通り消費。例: `v`→`w`→`w`→`d` ＝ 4ダメージ、`viwd` ＝ 3ダメージ。テキストオブジェクトの価値を体感させる

**レジスタ操作**
- `"a`〜`"z` — 名前付きレジスタへのヤンク・ペースト（例: `"ayiw`, `"ap`）
- `"0` — ヤンク専用レジスタ（削除で上書きされない）
- スキルツリー後半で解禁。レジスタの中身は手札エリアに「レジスタカード」として表示される（スキルカードとの統合UI）。`"ayiw`すると手札にカードが追加され、`"ap`で手札から出す。Vimの「レジスタは見えない」という難しさをゲームUIで可視化して解消する
- **レジスタカードUI**: カード領域の右端に独立グループとして配置。star色（`#ffd700`）の枠で通常カードと区別。カード上部にレジスタ名（`"a`）、下部に中身プレビュー（最大8文字、超過時省略）。ホバーで全文表示。`"ap`でカードが飛んでペースト、`"ayiw`で中身が更新される

**インサート系**
- `i` / `a` — カーソル前/後にインサート
- `I` / `A` — 行頭/行末にインサート
- `o` / `O` — 下/上に新行

**その他**
- `u` — アンドゥ
- `Ctrl+R` — リドゥ（undoした操作をやり直す）。`preventDefault()`でブラウザリロードを奪取
- `.` — ドットリピート（スキルツリー中盤以降で解禁）
- `Ctrl+d` / `Ctrl+u` — 半ページ下/上スクロール。スキルツリー後半で解禁、長文ファイルのステージと合わせて導入
- `/` + 検索語 + `Enter` — ファイル内検索。スキルツリー後半で解禁、長文ファイルのステージと合わせて導入
- `n` / `N` — 次/前の検索マッチに移動
- `{count}` + コマンド — 数値プレフィクス（`3w`, `5j`, `2dd`等）。カウントに関係なく1ダメージ。スキルツリー中盤以降で解禁——解禁前に`j`連打の苦痛を体験させ、`5j`一発の快感を演出
- `%` — 対応する括弧へジャンプ（`()`, `{}`, `[]`）
- `*` / `#` — カーソル下の単語を前方/後方検索
- `J` — 現在行と次の行をスペース区切りで結合
- `~` — カーソル下の文字の大文字/小文字を切替
- `gu` / `gU` + 移動/テキストオブジェクト — 範囲の小文字/大文字変換（`guiw`, `gUiw`等）
- `>>` / `<<` — インデント/デデント。Visualモード中の`>` / `<`も対応
- `gj` / `gk` — 表示行単位の上下移動（折り返し行がある場合）
- `zz` / `zt` / `zb` — カーソル位置を画面中央/上端/下端にスクロール

## 6. ブラウザ最適化（キー衝突対策）

ブラウザ標準ショートカットとの競合を回避するため、以下の制約を設ける。

- **使用禁止キー**: `Ctrl+w`, `Ctrl+t`, `Ctrl+n`等、ブラウザがブロックするタブ/ウィンドウ操作系。`preventDefault()`でも奪取不可能。
- `v1.0採用キー`: シングルキーまたはShift組み合わせ（大文字）を基本とし、`Ctrl+v`（矩形選択）・`Ctrl+d`（半ページ下）・`Ctrl+u`（半ページ上）・`Ctrl+R`（redo）は`preventDefault()`で奪取して使用する（ゲームエリアフォーカス時のみ）。

## 7. 運用設計

- **完全アカウントレス**: 登録・ログイン不要。進捗はLocalStorageで完結。
- **LocalStorage保存対象**: ステージ完了状態、☆数、ベストダメージ、ヒント使用フラグ、コマンドアンロック状態、ミュート設定。具体的なJSONスキーマは実装時に決定。
- **ツリーに沿ったアンロック**: スキルツリーの前提ノードをクリアしないと次に進めない。学習順序を保証し、「渇望→報酬」の感情設計を確実に機能させる。中級者も序盤ステージを素早く突破して先に進めるため摩擦は小さい。
- **即時アクセス**: 各ステージにハッシュベースの固定URL（`/#/play/12`等）を割り当て、摩擦ゼロで開始可能。静的ホスティング（GitHub Pages等）でサーバー設定不要。
- **SPAルーティング**: History API + ハッシュベースルーティング。画面遷移はブラウザ履歴に反映され、ブラウザバックでTreeに戻る等の自然な挙動。
- **ステージ中のリロード/タブクローズ**: オートセーブなし。リロード時はステージ最初からやり直し。ステージは短い（数手〜十数手）ので大きな損失にならない。
- **デスクトップ専用**: 物理キーボード必須。モバイル対応は行わない。
- **画面サイズ対応（2段階ブロック）**:
  - 最小対応幅1024px、推奨1280〜1920px。コンテンツ領域は`max-width: 1400px`で中央寄せ（ウルトラワイド対策）。
  - 768〜1023px（非タッチ）: 狭画面ブロック（「画面を広げてください」メッセージ）
  - 768px未満 or タッチデバイス: モバイルブロック（Landing相当の紹介ページ + 「⌨️ PCでアクセスしてください」メッセージ + URLコピーボタン）。ゲーム本体は読み込まない
- **ローディング画面**: 背景`#1a1a2e`、中央にVimQuestテキストロゴ＋ナビゲーターキューブのCSS回転アニメーション。`index.html`のインラインCSS/HTMLで実装（JSロード前に表示）。通常1〜2秒。
- **キーボードフォーカス管理**: ゲームエリア外をクリックしてフォーカスが外れた場合、エディタ領域に半透明オーバーレイ「クリックして再開」を表示。クリックでフォーカス復帰。
- **音声ミュートトグル**: Play画面のTop barに🔇アイコン。ON/OFFのみ。設定はLocalStorageで記憶。音量スライダーはv1.1以降。
