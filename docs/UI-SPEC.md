## 2. 画面構成と遷移

### 2.1 画面一覧

| 画面       | ヘッダー              | 性質         | 説明                                       |
| ---------- | --------------------- | ------------ | ------------------------------------------ |
| Landing    | なし                  | 独立ページ   | 初回 or URL直アクセス時のみ表示。1画面完結 |
| Skill Tree | あり（ロゴ+進捗表示） | メインページ | ホーム画面的位置付け                       |
| Play       | なし                  | 独立ページ   | エディタの没入感優先                       |
| Result     | なし                  | オーバーレイ | ステージクリア時                           |
| Game Over  | なし                  | オーバーレイ | ライフ0時                                  |
| New Weapon | なし                  | オーバーレイ | 新コマンドアンロック時                     |

### 2.2 画面遷移フロー

```
Landing ──→ Tree ←──────────────────┐
              │                      │
              ▼                      │
            Play ──→ Result ─────→ Tree
              │        │
              │        ├──→ [internal] Weapon ──→ Tutorial（4ノードのみ）──→ Tree
              │        │
              │        └──→ Play（次ステージへ直接遷移OK）
              │
              └──→ GameOver ──→ Play（リトライ）
                      │
                      └──→ Tree

  === N01専用フロー（初回のみ） ===
  Landing ─→ N01-1 → N01-2 → N01-3 → N01-4 → N01-5 → Tree（初表示）
  ↑ ツリー非表示で連続プレイ。各ステージクリア後にNEXTボタンで次へ。
```

**遷移ルール:**

- Landingの再訪問導線は不要。Treeがホーム
- Result: 「もう一度」「次のステージへ→」「ツリーへ」の3択
- Result + NEW WEAPON: 「次のステージへ→」押下で内部的にWeapon画面を経由
- Weapon: チュートリアル対象ノード（4ノード: N15,N24,N32）では Weapon → Tutorial → Tree。それ以外は Weapon → Tree
- GameOver: 「リトライ」「ツリーに戻る」の2択
- **N01連続プレイ**: Landingから直接N01-1に入り、N01-5までツリー非表示で連続プレイ。N01完了後にツリー画面初表示

### 2.3 画面遷移アニメーション

| 遷移               | 演出                                                                |
| ------------------ | ------------------------------------------------------------------- |
| 基本（全遷移共通） | 暗転フェード（0.2秒）                                               |
| Play → Result      | 暗転中に `STAGE CLEAR` テキストを0.5秒表示 → リザルトへフェードイン |
| Play → GameOver    | 画面が赤フラッシュ → フェード                                       |
| その他             | 統一フェードのみ                                                    |

### 2.4 ステージ開始イントロ

Play画面表示時、エディタ中央にステージ情報を1.5秒表示。

| 要素     | 内容                                       |
| -------- | ------------------------------------------ |
| 上段     | ステージ名（`STAGE 12`）                   |
| 下段     | 目標テキスト（`ci" で属性値を書き換えろ`） |
| 表示時間 | 1.5秒                                      |
| スキップ | 任意キー押下 or クリックで即開始           |

## 4. Skill Tree 画面

### 4.1 レイアウト

```
┌─────────────────────────────────────────────────┐
│ VimQuest — Skill Tree    Cleared: 11/48  ★28/144│  ← ヘッダー
├─────────────────────────────────────────────────┤
│                                                 │
│  [hjkl] → [x] → [wbe] → [dw de db] → ...      │  ← ツリーキャンバス
│                    │                             │  （パン/ズーム可能）
│                    └→ [0 $] → ...               │
│                                                 │
│                                                 │
│                                          [🎯]   │  ← 右下フローティング
└─────────────────────────────────────────────────┘
```

### 4.2 ノード状態

| 状態                    | 見た目                           | ホバー                                            | クリック                              |
| ----------------------- | -------------------------------- | ------------------------------------------------- | ------------------------------------- |
| cleared（クリア済み）   | 通常表示。★表示あり              | ツールチップ（コマンド名・説明・★数・ステージ数） | Playへ遷移（リプレイ）                |
| available（プレイ可能） | ハイライト表示。枠線が光る       | ツールチップ（同上）                              | Playへ遷移                            |
| locked（未解放）        | opacity 0.35。コマンド名は見える | コマンド名のみ表示                                | 軽くシェイク（左右2回揺れ）+ ロック音 |

### 4.3 操作

| 操作                                  | 動作                                                  |
| ------------------------------------- | ----------------------------------------------------- |
| マウスドラッグ（左ボタン）            | パン（移動）                                          |
| マウスホイール / トラックパッドピンチ | ズーム                                                |
| 🎯ボタン押下                          | availableノード群を中心にスムーズパン+ズーム（0.3秒） |

### 4.4 表示ルール

- **初期表示**: 全ノードが収まるようにズームレベルを自動フィット
- **ズーム制限**: 最小 = 全体俯瞰（全ノード表示）、最大 = ノード詳細が見える程度

### 4.5 リプレイ仕様

- clearedノードクリックでリプレイ可能
- 同じ手札（そのステージ時点のカード）でプレイ
- ☆・BESTは更新される
- ヒント使用フラグはリセット（ヒントなしで挑戦し直せる）
- **レイアウト**: 左→右に進行、ブランチは下に伸びる構造
- **ミニマップ**: v1.0不要（60コマンド程度なら全体フィット可能）

### 5.1 レイアウト

```
┌─────────────────────────────────────────────────────┐
│ :q!  ♥━━━━━━━━━ 7/10   STAGE 12 — ci"で属性値を書き換えろ  NORMAL │ ← Top bar
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │ ● ● ● style.css                             │   │
│  │  1  .container {                             │   │  ← エディタ領域
│  │  2    display: flex;                         │   │    （画面の70%）
│  │  3    color: "█ff0000";                      │   │
│  │  4    font-size: 16px;                       │   │
│  │  5    margin: 0 auto;                        │   │
│  │  6  }                                        │   │
│  │                                               │   │
│  │──────────────────────────────────────────────│   │
│  │ 3d                                    入力中 │   │  ← ステータスライン
│  └─────────────────────────────────────────────┘   │
│                                              ◇     │  ← ナビゲーター（右下）
│                                             HINT   │
├─────────────────────────────────────────────────────┤
│                     HAND                            │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌────┐     │  ← カード領域
│  │ d│ │ c│ │ y│ │ f│ │ t│ │iw│ │i"│ │ci" │     │    （画面の30%）
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └────┘     │
└─────────────────────────────────────────────────────┘
```

### 5.2 エリア比率

- **エディタ : カード = 70 : 30** 固定比率
- コードが少ないステージでもカード領域が跳ね上がらないよう固定

### 5.3 Top bar

| 要素             | 位置      | 説明                                                           |
| ---------------- | --------- | -------------------------------------------------------------- |
| `:q!` ボタン     | 左端      | テキストボタン。クリックでツリーに戻る（確認なし即時遷移）     |
| `:r` ボタン      | `:q!`の右 | テキストボタン。クリックでステージをリトライ（初期状態に戻す） |
| 🔇 ミュート      | `:r`の右  | 音声ON/OFFトグル。設定はLocalStorageで記憶                     |
| ライフゲージ     | 左寄り    | `♥━━━━━━━━ 7/10` のバー+テキスト                               |
| ステージタイトル | 中央      | `STAGE 12 — ci" で属性値を書き換えろ`                          |
| モード表示       | 右寄り    | `NORMAL` / `INSERT` / `VISUAL`                                 |

### 5.4 エディタ領域

#### カーソル表示

| モード | カーソル形状                  | 背景色       |
| ------ | ----------------------------- | ------------ |
| Normal | ブロック（█）                 | `#1a1a2e`    |
| Insert | ライン（｜）                  | `#2e1a1a`    |
| Visual | ブロック + 選択範囲ハイライト | 紫 `#2a1a3e` |

- **点滅**: なし。代わりにカーソルに微かなグロー（発光）で位置を強調
- **Visual選択範囲**: `rgba(124,124,255,0.3)`（accent半透明）で塗る。カーソル位置は `rgba(124,124,255,0.6)` で強調

#### ステータスライン

- エディタ下部に1行
- 入力中のキーシーケンスをリアルタイム表示（`3`, `3d`, `3dw` 等）
- 実Vimのフィードバックを学ぶ教育的役割
- 不正コマンド時: `(unknown)` を表示
- undo時: `u` 表示後、巻き戻されたコマンドを取り消し線で一瞬表示

### 5.5 カード領域（HAND）

#### 基礎コマンド行（base行）

N01クリア後、手札パネル上部に `BASE_COMMANDS`（h j k l w b e x i a）を小さなバッジで表示。

| 項目 | 値 |
|------|-----|
| フォントサイズ | 11px（通常カードの14pxより小さい） |
| 背景 | `rgba(34, 34, 68, 0.5)` |
| ボーダー | `1px solid rgba(68, 68, 120, 0.4)` |
| パディング | `2px 8px` |
| 角丸 | 4px |
| ホバーエフェクト | なし（注目させない） |

**表示条件:**
- N01ステージ中: base行非表示（全コマンドがavailableCommandsとして大きなカードで表示）
- N01クリア後: base行表示開始
- Visualモード時: v/Vをbase行から除外
- Insertモード時: base行も非表示（現行通り）

```
N01ステージ:
┌────────────────────────────────────┐
│ HAND                               │
│ [h] [l]                           │  ← availableCommandsのみ（大きいカード）
└────────────────────────────────────┘

N01以降のステージ:
┌────────────────────────────────────┐
│ HAND                               │
│  h  j  k  l  w  b  e  x  i  a    │  ← base行（小さなバッジ）
│ [dw] [de] [db]                    │  ← availableCommands（大きいカード）
└────────────────────────────────────┘
```

#### 表示ルール

- コマンド名のみ大きく表示（例: `d`, `ciw`, `f`）
- ラベル（delete, change等）はホバーツールチップで表示
- 最大12〜15枚想定
- 多い場合は2行表示:
  - **上段**: 動詞・モーション（`d`, `c`, `y`, `f`, `t` 等）
  - **下段**: テキストオブジェクト・合成（`iw`, `i"`, `a(`, `ci"` 等）

#### コマンド入力フィードバック（カード連動）

| 入力状態              | カードUI                                                                                                                          |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| 待機中                | 全カード通常表示                                                                                                                  |
| カウント入力（`3`）   | 対応カード上に `×3` バッジ表示                                                                                                    |
| オペレータ入力（`d`） | `d` カードが浮き上がり光る。有効な対象カード（`w`, `iw`, `$` 等）がアクティブ化。無効カード（他の動詞 `c`, `y` 等）がグレーアウト |
| 複合入力（`3d`）      | `d` 浮上+光り + 対象カードアクティブ + `×3` バッジ                                                                                |

#### カード合成演出（`d` + `iw` → `diw`）

```
Step 1: `d` 押下
  → d カードがパルス（光って維持）
  → 有効な対象カードがアクティブ化

Step 2: `iw` 押下
  → d カードのラベルが `diw` に変化
  → iw カードがフェードアウト

Step 3: 効果適用
  → diw カードがエディタに向かって飛ぶ
  → テキストに効果適用 + 打鍵音再生
```

- **コンセプト**: 「動詞が対象を食べて合成コマンドになる」の可視化
- **速度**: 全体 0.3〜0.5秒
- **undo時**: 逆再生（合成カード → 分解して元の2枚に戻る、0.2秒）

### 5.6 不正コマンド入力

手札にないキーや現在のモードで無効なキーを押した場合:

- 画面端が一瞬赤フラッシュ（vignette的エフェクト）
- ステータスラインに `(unknown)` 表示
- 不発音（通常の打鍵音とは異なる短い音）
- **ライフ消費なし、ダメージカウントなし**

### 5.7 Insertモード時のカード領域

Insertモード中はスキルカードの代わりにInsert情報を表示。

| 表示要素        | 説明                |
| --------------- | ------------------- |
| 入力中表示      | 「入力中…」テキスト |
| Escリマインダー | 「Esc で確定」      |

- シンプルな表示でInsertモード中であることを示す
- 実際のダメージ計算は内部で文字数を追跡（5文字無料、以降5文字ごと+1）

### 5.8 検索（`/`）UI

カード領域下部のパーサーバッファ表示を検索入力欄として利用。

| 要素       | 動作                                          |
| ---------- | --------------------------------------------- |
| `/` 押下   | パーサーバッファが検索入力欄に変化（`/█`）    |
| 検索語入力 | パーサーバッファにパターン累積表示（`/abc█`） |
| Enter      | 検索確定、最初のマッチにジャンプ              |
| `n` / `N`  | 次/前のマッチに移動                           |

**マッチハイライト:**

- 全マッチ: `rgba(255,215,0,0.3)`（star色半透明）
- 現在マッチ: より濃いstar色で強調

### 5.9 ビジョン（透視）システム

Space キーのホールドでゴールテキストを透視する。

| 操作       | 動作                                                                      |
| ---------- | ------------------------------------------------------------------------- |
| Space 押下 | 即座にゴールテキストに切替。変更箇所を0.2秒 success色フラッシュで視線誘導 |
| Space 離す | 即座に現在テキストに戻る（フラッシュなし）                                |

- 頻繁に使うため、重い演出は避ける
- 差分ハイライトにより「どこを変えればいいか」の気づきを促進

### 5.10 Undo / Redo

`u` は正当なVim技術として扱う。**無料・無制限**。

| 操作             | 動作                                 |
| ---------------- | ------------------------------------ |
| `u`（undo）      | テキスト+ライフを1手前に完全巻き戻し |
| `Ctrl+R`（redo） | undoした操作をやり直す               |

**undo時のUI挙動:**

| 要素             | 演出                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------- |
| テキスト         | 1手前に巻き戻り + 変更箇所 success色フラッシュ                                        |
| ライフゲージ     | 消費分が回復（ゲージが伸びるアニメ）                                                  |
| カード           | 合成カードは分解して元の2枚に戻る（合成の逆再生、0.2秒）                              |
| ステータスライン | `u` 表示後、巻き戻されたコマンドを取り消し線で一瞬表示                                |
| SPELL記録        | undoされた操作はSPELLから消える。redo（`Ctrl+R`）で復活。クリア時点の操作列が最終記録 |

- 連続undo可能（初期状態まで全て戻せる）
- SPELLには「磨き上げた解法」のみが残る

### 5.11 ナビゲーター（浮遊キューブ）

右下隅の幾何学的な浮遊キューブ。操作品質の感情フィードバック装置。

#### v1.0 最小セット

| トリガー     | リアクション              |
| ------------ | ------------------------- |
| 常時         | ゆっくり回転              |
| 効率的な操作 | 一瞬発光                  |
| waste操作    | 一瞬暗くなる              |
| ライフ低下   | 回転が遅くなる + 暗くなる |

- クリックでヒント（初期状態からの☆3解法オートプレイ）をオーバーレイ表示。同一プレイ中に何度でも再生可能（☆1確定は初回使用で決定済み）
- オーバーレイ表示のためモード（Normal/Insert/Visual）に関係なくいつでも再生可能。現在のエディタ状態はリセットされない
- ヒントオーバーレイ表示中はキャプチャフェーズで全キーをブロック。Escのみで閉じる（背後のゲーム画面にキーが伝播しない）
- チュートリアル画面のナビゲーターバーでも同じミニ3D回転キューブ（CSS）をアイコンとして使用（プレイ画面と統一）
- 細かいリアクション（undo逆回転、クリア時激しく輝く等）はv1.1以降
- プレイテストで調整

### 5.12 ドットリピート（`.`）のカード演出

`.`カードは手札に常時表示。

| 操作   | 動作                                                             |
| ------ | ---------------------------------------------------------------- |
| ホバー | 「前回: ciw」のように直前コマンドをツールチップ表示              |
| 押下   | 前回コマンドのカード合成演出をゴースト（半透明）で0.15秒高速再生 |

- 「何が繰り返されたか」を視覚的に伝える教育的演出
- 0.15秒の高速再生でプレイの邪魔にならない

### 5.14 ギブアップ・リトライ導線

- 左上の `:q!` ボタン（テキスト）でツリーに戻る。確認ダイアログなしの即時遷移（Vimの`:q!`=強制終了の意味を体現）
- `:q!` をキーボードで入力してもツリーに戻れる（イースターエッグ）
- `Esc` キーでもツリーに戻れる（パーサーがアイドル状態かつNORMALモード時のみ）
- 左上の `:r` ボタン（テキスト）でリトライ。ステージを初期状態に戻す
- `:r` をキーボードで入力してもリトライ可能

### 5.15 キーボードイベントマッピング

| ブラウザイベント     | マッピング先   | 備考                                          |
| -------------------- | -------------- | --------------------------------------------- |
| `Ctrl+r`             | `Ctrl+R`       |                                               |
| `Ctrl+d`             | `Ctrl+d`       |                                               |
| `Ctrl+u`             | `Ctrl+u`       |                                               |
| `Ctrl+v`             | `Ctrl+v`       | `preventDefault()` でブラウザのペーストを抑止 |
| `Escape`             | `Esc`          |                                               |
| `Backspace`          | `Backspace`    |                                               |
| `Enter`              | `Enter`        |                                               |

### 5.16 検索モードとコロンバッファの競合回避

`/` 検索入力中はパーサーバッファが `/pattern` 形式になる。この状態で `:` を入力してもコロンモード（`:q!`/`:r`）には入らず、検索パターンの一部として処理される。
