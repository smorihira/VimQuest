# チュートリアル拡充計画

## 現状

- N01 のみ全10ステージにチュートリアルあり（19コマンド網羅）
- N02〜N31（29ノード）は **チュートリアルゼロ**
- 特にマルチコマンドノード（N13: D/C/S、N15: f/t/;/,、N20: 14種のTextObj 等）で新コマンドの違いが一切説明されないのが問題

## 方針

1. **全ノードの Teach ステージにチュートリアルを付ける**（最低限）
2. コマンドの性質が近い場合 → 1つのチュートリアルにまとめて OK
3. コマンドの性質が異なる場合 → ステージを分けてそれぞれにチュートリアルを付ける
4. Practice/Challenge にはチュートリアル不要（Teach で学んだコマンドの実践）

---

## アーキテクチャ変更: チュートリアル ↔ ステージ統合

### 現状の問題

```
[チュートリアル]            [ステージ]
独自のテキスト "hello!"  →  別のテキスト "foo bar baz"
独自のカーソル位置        →  別のカーソル位置
ステップ完了 → 完了画面   →  ステージが最初からスタート（断絶）
```

- チュートリアルとステージが **完全に別世界**
- チュートリアルで練習した内容とステージの課題に連続性がない
- 「さっきやったことをもう1回やるだけ」感が出てしまう

### 新設計: 「ガイド付き前半 → 自力で後半」

```
[ステージ開始]
同じテキスト・同じゴール
  ├── 前半: チュートリアルがステップごとにガイド（操作を教える）
  └── 後半: チュートリアル終了 → そのままプレイに移行（エディタ状態を引き継ぎ）
              プレイヤーが自力でゴールまで完成させる
```

### 必要な変更

#### 1. Tutorial 型の変更

```typescript
// Before
export interface Tutorial {
  nodeId: string
  stageId?: string
  initialSetup: { text: string; cursor: CursorPosition }  // ← 独自の初期状態
  steps: TutorialStep[]
}

// After
export interface Tutorial {
  nodeId: string
  stageId: string           // ← 必須に（ステージと1:1対応）
  initialSetup?: { ... }    // ← オプショナル維持。あれば旧方式（N01-1用）
  steps: TutorialStep[]
}
```

#### 2. StageTutorial コンポーネントの変更

- `tutorial.initialSetup` の代わりに `stage.initialText` + `stage.initialCursor` で初期化
- チュートリアル完了時に **エディタの現在状態を返す**（テキスト・カーソル・レジスタ等）

```typescript
onComplete={(status, editorState) => {
  // editorState をそのまま PlayScreenInner に渡す
}}
```

#### 3. PlayScreen の変更

- チュートリアル完了後、**ステージを最初からではなく**途中状態から開始
- `editorState` を `PlayScreenInner` の初期状態として注入

```typescript
// PlayScreenInner に途中状態を渡す
<PlayScreenInner
  stage={stage}
  initialOverride={editorState}  // チュートリアルの続きから
/>
```

#### 4. ステージ設計の変更

- Teach ステージは **チュートリアルで解く分 + 自力で解く分** を含む長さにする
- 例: 6手で解けるステージ → 前半3手をチュートリアル、後半3手を自力

### 設計例: N15 (f コマンド)

```
ステージ初期テキスト:
  "find the fox and fix the flag"
  カーソル: (0, 0)

ゴール:
  "find the fox and fix the flag"
  カーソル: (0, 27)  ← 'g' の位置

チュートリアル前半（ガイド付き）:
  Step 1: "f を押してみろ。f{文字} でその文字にジャンプする"
          expectedKey: 'f'
  Step 2: "o を押せ。'fox' の 'o' に飛ぶぞ"
          expectedKey: 'o'
  Step 3: "次は t を試せ。t{文字} はその文字の手前に止まる"
          expectedKey: 't'
  Step 4: "x を押せ。'fix' の 'x' の手前に止まるはずだ"
          expectedKey: 'x'
  Step 5: "f と t の違いが分かったな。残りは自分で 'g' まで辿り着け"
          expectedKey: null  ← 情報ステップ（任意キーで終了）

プレイ後半（自力）:
  現在位置から 'g' まで f/t を使って移動
  → ダメージ計測・☆判定はこの後半部分のみ？ or 全体？
```

### 決定事項

- [x] **ダメージ計測の範囲**: 後半（自力パート）のみ計測
  - チュートリアル部分は安全地帯。チュートリアル完了時にダメージを0リセットして後半開始
- [x] **既存N01チュートリアルの移行**: N01-1のみ旧方式維持、N01-2以降は新方式に移行
  - N01-1はVimQuestの操作方法（Space長押し、:h、:e!、📖ボタン等）を教えるオンボーディング
  - N01-2以降は純粋にVimコマンドを教えるだけなので新方式に統一
  - `initialSetup` があれば旧方式フォールバック、なければステージデータを使用
- [x] **チュートリアル→プレイ移行時の演出**: ナビゲーターの一言で移行
  - 最後のステップを `expectedKey: null`（情報ステップ）にして「残りは自分でやれ」メッセージ
  - 任意キーで閉じるとそのままプレイモードに切り替わる（UIの変化は吹き出しが消えるだけ）
- [x] **スタンドアロン復習モード**: ステージデータを読み込んで再生
  - 復習時も `stage.initialText` + `stage.initialCursor` からスタート
  - チュートリアル完了後は「プレイに進む」か「閉じる」を選択

## 🔴 要検討ノード 詳細

### N10: d+motion (dw, de, db)

- **現状**: T(dw)/P(de+db)/C(混合)
- **案A**: T のチュートリアルで dw/de/db の違いをまとめて説明
- **案B**: T を dw、新ステージを de/db にして2チュートリアル
- **推奨**: 案A — 3つとも「d+移動」の派生で、違いは削除範囲のみ

### N13: ショートカット (D, C, S)

- **現状**: T(D) / P(S, C)
- **案A**: T のチュートリアルで D→C→S を順に説明
- **案B**: D / C / S それぞれ独立ステージ（3ステージ、各チュートリアル付き）
- **案C**: T(D) + 新ステージ(C) + 新ステージ(S) の3Teach + 既存P,C維持
- **推奨**: 案C — D=d$, C=c$, S=cc と元の操作が異なり混乱しやすい

### N15: 行内ジャンプ (f, t, ;, ,)

- **現状**: T(f) / P(;) / C(混合)
- **案A**: T で f/t/;/, すべて説明
- **案B**: T(f/t) + P(;/,) — 移動と繰り返しを分離
- **推奨**: 案B — f/t は類似（到達 vs 手前）、;/, は別概念（リピート）

### N20: デリミタTextObj (14種)

- **現状**: T(ci", di") / P(ci(, da()
- **案A**: T で i/a の概念 + 代表例(" と ()をチュートリアル
- **案B**: T(引用符系) + 新ステージ(括弧系) の2チュートリアル
- **推奨**: 案A — i/a の概念理解が本質。デリミタの種類は同じパターンの適用

### N21: d+TextObj (diw, di", da()

- **現状**: T(diw, di") / P(da()
- **案A**: T のチュートリアルでまとめて説明
- **推奨**: 案A — d+TextObj はN19/N20の延長で、概念は既知

### N22: c+TextObj (ciw, ci", ca()

- **現状**: T(ciw) / P(ci") / C(ca()
- **案A**: T のチュートリアルでまとめて説明
- **推奨**: 案A — c+TextObj もN19/N20の延長

### N27: Visualモード (v, V)

- **現状**: T(v+d) / P(V+d, f/t)
- **案A**: T で v/V 両方説明
- **案B**: T(v) + 新ステージ(V) の2チュートリアル
- **推奨**: 案A — v=文字/V=行 のシンプルな対比で1チュートリアルに収まる

### N29: ヤンク＆ペースト (y, p, P)

- **現状**: T(yiw+p) / P(yy+p) / C(dd+P)
- **案A**: T のチュートリアルで y→p→P の流れを説明
- **推奨**: 案A — y(コピー)+p/P(貼り付け)はセットで学ぶのが自然

### N31: レジスタ ("a, "0)

- **現状**: T("ay+p) / P("a, "b, "0)
- **案A**: T のチュートリアルで名前付きレジスタの概念を説明
- **推奨**: 案A — "a と "0 は同じ仕組み（レジスタ指定）のバリエーション
