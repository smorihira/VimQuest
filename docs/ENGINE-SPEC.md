# VimQuest エンジン仕様書

ゲームエンジン内部の動作仕様。UI仕様（UI-SPEC.md）とプロダクト仕様（VimQuest.md）を補完し、実装に必要なアルゴリズム・データ構造・エッジケースを定義する。

> 本ドキュメントの各項目は ENGINE-REVIEW.md の議論を経て決定されたもの。

## 1. テキスト比較アルゴリズム（ステージクリア判定）

### 1.1 基本ルール

ステージクリアは **`bufferText === goalText` の厳密な文字列比較** で判定する。

| 項目 | 仕様 |
|------|------|
| 比較方式 | JavaScript の `===`（厳密等価） |
| 改行コード | LF（`\n`）固定。ステージデータもLF統一 |
| 末尾空白 | 比較対象に含む（空白も一致必須） |
| 最終行の末尾改行 | なし（最終行は `\n` で終わらない） |
| Unicode正規化 | なし。バイト列一致 |
| 比較対象 | テキスト内容のみ（カーソル位置はセクション6の拡張条件で別途判定） |

### 1.2 判定タイミング

- **コマンド実行ごと**にリアルタイム判定
- Normalモードでの編集コマンド実行後、Insertモードで `Esc` 押下後に判定
- 拡張条件（セクション6）がある場合はテキスト一致 AND 拡張条件すべてを満たした時点でクリア

### 1.3 ステージデータの責任

ゴールテキストの正確な定義はステージデータ側の責任。エンジンは正規化・トリムを一切行わない。

## 2. コマンド分類とカード合成ルール

### 2.1 コマンドカテゴリ

全コマンドを以下の4カテゴリに分類する。

#### オペレータ（Operator）
モーションまたはテキストオブジェクトを待ち、範囲を決定してから実行する。

| コマンド | 説明 |
|---------|------|
| `d` | 削除 |
| `c` | 変更（削除+Insert） |
| `y` | ヤンク（コピー） |
| `gu` | 小文字化 |
| `gU` | 大文字化 |
| `>` | インデント |
| `<` | デデント |

#### モーション（Motion）
カーソルを移動する。単独で使用可能。オペレータの対象にもなれる。

| コマンド | 説明 |
|---------|------|
| `h` `j` `k` `l` | 基本移動 |
| `w` `b` `e` | 単語移動 |
| `W` `B` `E` | WORD移動 |
| `0` `$` `^` | 行頭/行末/非空白先頭 |
| `gg` `G` `{n}G` | ファイル先頭/末尾/指定行 |
| `f{c}` `t{c}` | 行内ジャンプ（指定文字まで/手前） |
| `;` `,` | f/tの繰り返し（同方向/逆方向） |
| `%` | 対応括弧ジャンプ |
| `/pattern` + `Enter` | 検索ジャンプ |
| `n` `N` | 次/前の検索マッチ |
| `*` `#` | カーソル下単語の前方/後方検索 |
| `gj` `gk` | 表示行移動 |
| `Ctrl+d` `Ctrl+u` | 半ページスクロール |

#### テキストオブジェクト（Text Object）
オペレータの対象としてのみ使用可能。Visualモード中は選択範囲として単独使用可。

| コマンド | 説明 |
|---------|------|
| `iw` / `aw` | inner/around word |
| `i"` / `a"` | inner/around ダブルクォート |
| `i'` / `a'` | inner/around シングルクォート |
| `i(` / `a(` | inner/around 括弧 |
| `i{` / `a{` | inner/around 波括弧 |
| `i[` / `a[` | inner/around 角括弧 |
| `i<` / `a<` | inner/around 山括弧 |
| `it` / `at` | inner/around HTMLタグ |

#### 即時コマンド（Instant）
単独で完結する。モーション/オペレータと組み合わせない。

| コマンド | 説明 |
|---------|------|
| `x` | 1文字削除 |
| `r{c}` | 1文字置換 |
| `s` | 1文字削除+Insert（`cl`相当） |
| `S` | 行削除+Insert（`cc`相当） |
| `C` | 行末まで変更（`c$`相当） |
| `D` | 行末まで削除（`d$`相当） |
| `i` `a` `I` `A` | Insert開始（各種） |
| `o` `O` | 新行+Insert |
| `p` `P` | ペースト |
| `J` | 行結合 |
| `~` | 大文字小文字切替 |
| `u` | undo |
| `Ctrl+R` | redo |
| `.` | ドットリピート |
| `v` `V` `Ctrl+v` | Visualモード開始 |
| `dd` `cc` `yy` | 行操作（オペレータ×2の特殊形） |
| `>>` `<<` | 行インデント/デデント |
| `zz` `zt` `zb` | 画面スクロール位置調整 |

### 2.2 組み合わせルール

| パターン | 有効性 | 例 |
|---------|--------|-----|
| オペレータ + モーション | ✅ 有効 | `dw`, `c$`, `ygg` |
| オペレータ + テキストオブジェクト | ✅ 有効 | `diw`, `ci"`, `ya(` |
| オペレータ × 2（同キー） | ✅ 行操作 | `dd`, `cc`, `yy`, `>>`, `<<`, `guu`, `gUU` |
| オペレータ + オペレータ（異キー） | ❌ 無効 | `dc`, `cy` → エラー |
| テキストオブジェクト単独（Normal） | ❌ 無効 | `iw` 単体 → エラー |
| テキストオブジェクト単独（Visual） | ✅ 選択範囲 | `viw`, `vi"` |
| 数値プレフィックス + 任意 | ✅ 付与可 | `3dw`, `2j`, `d3w` |

### 2.3 カードUI連動

- オペレータカード押下 → 有効な対象（モーション/テキストオブジェクト）カードがハイライト
- 無効なカードは暗いまま、入力を受け付けない
- 数値入力中はカード上に `×N` バッジを表示
- `f`/`t` 押下後は文字入力待ち状態（カードは全て非アクティブ、ステータスラインに `f` 表示）

## 3. 検索エンジン仕様

### 3.1 基本仕様

| 項目 | 仕様 |
|------|------|
| マッチ方式 | **リテラル文字列**（正規表現なし） |
| 大文字小文字 | **区別する**（case-sensitive） |
| ラップアラウンド | **あり**（EOF到達後、先頭に戻って検索続行） |
| 検索方向 | `/` = 前方（カーソル位置以降）、`n` = 同方向、`N` = 逆方向 |

### 3.2 UI

| 状態 | 表示 |
|------|------|
| `/`入力中 | ステータスラインが検索入力欄に変化。パターンをリアルタイム表示 |
| Enter確定後 | マッチ箇所をstar色半透明（`rgba(255,215,0,0.3)`）でハイライト。現在マッチはより濃い色 |
| リアルタイムハイライト | **なし**（v1.0。Enter確定後に一括ハイライト） |

### 3.3 エラー処理

| シナリオ | 挙動 |
|---------|------|
| マッチゼロ | ステータスラインに `(pattern not found)` 表示。**ダメージなし** |
| 空パターンで確定 | 何もしない。ダメージなし |
| `Esc`でキャンセル | 検索中止。ダメージなし |

### 3.4 ダメージ

| 操作 | ダメージ |
|------|---------|
| `/pattern` + `Enter`（マッチあり） | 1 |
| `/pattern` + `Enter`（マッチなし） | 0 |
| `n` / `N` | 1 |
| `*` / `#` | 1 |

### 3.5 オペレータ対象

検索はモーションカテゴリに属するため、オペレータの対象として使用可能。

```
d/pattern  → カーソル位置から最初のマッチ開始位置まで削除
c/pattern  → 同上 + Insert開始
y/pattern  → 同上をヤンク
```

## 4. undo/redo/SPELL 状態遷移

### 4.1 undoスタックモデル

標準エディタ準拠のundoツリーを採用。

```
undoStack: [op1, op2, op3, ...]  ← 操作を順番に積む
redoStack: [op4, op5, ...]       ← undoで取り出した操作を積む
```

### 4.2 状態遷移ルール

| アクション | undoStack | redoStack | ダメージ |
|-----------|-----------|-----------|---------|
| コマンド実行 | push(op) | **クリア** | +1（コマンドに応じて） |
| `u`（undo） | pop → redoへ | push(op) | 該当操作分を復元 |
| `Ctrl+R`（redo） | push(op) | pop | 該当操作分を再消費 |
| undo後に新コマンド | push(newOp) | **クリア** | +1 |

### 4.3 SPELL記録ルール

| ルール | 説明 |
|--------|------|
| 記録対象 | クリア時点のundoスタック上の操作列 |
| undo自体 | SPELLに**記録しない**（メタ操作） |
| redo自体 | SPELLに**記録しない**（メタ操作） |
| undoされた操作 | SPELLから**消える** |
| redoされた操作 | SPELLに**復活する** |

### 4.4 具体例

#### 例1: undo→新コマンド
```
操作: dw → dd → u → u → cw → "hello" → Esc

ステップ:
  1. dw      undoStack=[dw]           redoStack=[]        damage=1
  2. dd      undoStack=[dw,dd]        redoStack=[]        damage=2
  3. u       undoStack=[dw]           redoStack=[dd]      damage=1
  4. u       undoStack=[]             redoStack=[dw,dd]   damage=0
  5. cw+hello undoStack=[cw+"hello"]  redoStack=[]        damage=1

SPELL: [cw → "hello"]
最終ダメージ: 1
```

#### 例2: undo→redo
```
操作: dw → u → Ctrl+R

ステップ:
  1. dw      undoStack=[dw]    redoStack=[]    damage=1
  2. u       undoStack=[]      redoStack=[dw]  damage=0
  3. Ctrl+R  undoStack=[dw]    redoStack=[]    damage=1

SPELL: [dw]
最終ダメージ: 1
```

#### 例3: 複数undo→途中からやり直し
```
操作: dw → dd → x → u → u → ciw → "new" → Esc

ステップ:
  1. dw         undoStack=[dw]              damage=1
  2. dd         undoStack=[dw,dd]           damage=2
  3. x          undoStack=[dw,dd,x]         damage=3
  4. u          undoStack=[dw,dd]           damage=2
  5. u          undoStack=[dw]              damage=1
  6. ciw+"new"  undoStack=[dw,ciw+"new"]    damage=2  ← redoStack=[dd,x]はクリア

SPELL: [dw, ciw → "new"]
最終ダメージ: 2
```

## 5. Insertモード ダメージ計算

### 5.1 計算式

```
Insert系コマンドのダメージ = 1（進入コスト） + floor((max(charCount - 5, 0)) / 5)
```

- `charCount`: Esc押下時点のInsert中の最終入力文字数
- バックスペース: 文字数カウントから減算（`hello`→BS→`hell` = 4文字）
- Enter: 1文字としてカウント（改行 = 1文字）
- モード即キャンセル（`i`→即`Esc`）: 進入コストも取消し（0ダメージ）

### 5.2 オペレータ合成の場合

`ciw`, `cf{c}` 等のオペレータ合成でInsertに入る場合も、**合成全体で1ダメージ**（進入コスト）。入力文字のコストは同じルールで追加。

### 5.3 計算例テーブル

| 操作 | charCount | 内訳 | ダメージ |
|------|-----------|------|---------|
| `i` + `hello`(5) + `Esc` | 5 | 1 + floor(0/5) = 1 | **1** |
| `i` + `hello!`(6) + `Esc` | 6 | 1 + floor(1/5) = 1+1 | **2** |
| `i` + 11文字 + `Esc` | 11 | 1 + floor(6/5) = 1+2 | **3** |
| `ciw` + `replace`(7) + `Esc` | 7 | 1 + floor(2/5) = 1+1 | **2** |
| `s` + `abc`(3) + `Esc` | 3 | 1 + floor(0/5) = 1 | **1** |
| `O` + 0文字 + `Esc` | 0 | 1 + 0 = 1 | **1** |
| `o` + `Enter`(1) + `abc`(3) + `Esc` | 4 | 1 + floor(0/5) = 1 | **1** |
| `A` + 20文字 + `Esc` | 20 | 1 + floor(15/5) = 1+3 | **4** |
| `i` → 即`Esc` | 0 | モード即キャンセル | **0** |

### 5.4 Visual Block I/A の特則

`Ctrl+v` → `I` or `A` でのテキスト入力は **実タイプ量ベース**。

```
例: 3文字 × 5行 → charCount = 3（実際にタイプした文字数）
ダメージ: 1 + floor(0/5) = 1
```

## 6. ステージデータスキーマ

### 6.1 基本スキーマ

```json
{
  "id": 12,
  "title": "ci\" で属性値を書き換えろ",
  "language": "css",
  "initial": ".container {\n  display: flex;\n  color: \"#ff0000\";\n  font-size: 16px;\n}",
  "goal": ".container {\n  display: flex;\n  color: \"#333333\";\n  font-size: 16px;\n}",
  "initialCursor": { "line": 2, "col": 10 },
  "life": 10,
  "stars": [3, 5, 8],
  "cards": ["d", "c", "y", "f", "t", "iw", "i\"", "ci\""],
  "optimalSpell": ["3j", "ci\"", "#333333", "Esc"],
  "clearConditions": {
    "text": true
  },
  "unlocks": ["s", "S", "C"]
}
```

### 6.2 フィールド定義

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `id` | number | ✅ | ステージ固有ID |
| `title` | string | ✅ | ステージタイトル |
| `language` | string | ✅ | シンタックスハイライト言語。有効値: `"css"`, `"html"`, `"javascript"`, `"json"`, `"markdown"`, `"python"`, `"plaintext"` |
| `initial` | string | ✅ | 初期テキスト（LF改行） |
| `goal` | string | ✅ | ゴールテキスト（LF改行） |
| `initialCursor` | `{line, col}` | ✅ | 初期カーソル位置（0-indexed） |
| `life` | number | ✅ | ライフ総量 |
| `stars` | `[number, number, number]` | ✅ | ☆3/☆2/☆1のダメージ閾値。`[3, 5, 8]` = 3ダメージ以下で☆3、5以下で☆2、8以下で☆1 |
| `cards` | string[] | ✅ | 使用可能コマンド（手札）のリスト |
| `optimalSpell` | string[] | ✅ | ☆3解法のコマンド列（ヒント/オートプレイ用） |
| `clearConditions` | object | ❌ | 拡張クリア条件。省略時は `{ "text": true }` |
| `unlocks` | string[] | ❌ | クリアでアンロックされるコマンド。省略時はアンロックなし |

### 6.3 拡張クリア条件（clearConditions）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `text` | boolean | テキスト一致判定（デフォルト `true`） |
| `cursor` | `{line: number, col: number}` | クリア時のカーソル位置（0-indexed）。省略時はカーソル位置不問 |
| `registers` | `{[name]: string}` | 指定レジスタの中身が一致していること。省略時はレジスタ不問 |

- 複数条件は **すべてAND** で評価
- 拡張条件があるステージはイントロで条件をアイコン提示（🎯カーソル位置、📋レジスタ）

```json
// 例: テキスト一致 + カーソル位置 + レジスタ
{
  "clearConditions": {
    "text": true,
    "cursor": { "line": 0, "col": 12 },
    "registers": { "a": "hello" }
  }
}
```

## 7. LocalStorage 永続化

### 7.1 保存対象

| キー | 内容 |
|------|------|
| ステージ完了状態 | 各ステージのクリア有無 |
| ☆数 | 各ステージのベスト☆評価 |
| ベストダメージ | 各ステージの最少ダメージ記録 |
| ヒント使用フラグ | 各ステージでヒントを使用したか |
| コマンドアンロック状態 | 解放済みコマンドリスト |
| チュートリアル状態 | 各ノードのチュートリアル完了/スキップ状態 |
| ミュート設定 | ON/OFF |
| データバージョン | マイグレーション用 |

### 7.2 エラーハンドリング

| シナリオ | 対応 |
|---------|------|
| **LocalStorage無効**（プライベートブラウジング等） | 起動時に `localStorage.setItem` テスト実行。失敗→インメモリフォールバック。Landing画面に「⚠ 進捗は保存されません」バナー表示 |
| **容量超過**（`QuotaExceededError`） | `setItem` の例外をcatch。古いデータ（個別ステージのSPELL記録等）を削除して再試行。それでも失敗→バナー警告 |
| **JSON破損** | `JSON.parse` で例外→該当キーを削除して初期値にリセット。全損の場合は全データリセット+「データが破損したため初期化しました」通知 |
| **バージョン不一致** | 保存データに `version` フィールドを持たせる。読み込み時にバージョンチェック→マイグレーション関数で変換。変換不能→全リセット |

### 7.3 設計方針

**データ喪失よりアプリ継続を優先**。壊れた場合は潔くリセット。ゲームデータは「時間投資」であり「金銭投資」ではないため許容範囲。

## 8. 追加仕様（旧DEFERRED → 確定）

### 8.1 モード背景色

| モード | hex値 |
|--------|-------|
| Normal | `#1a1a2e`（ベース背景色と同一。デフォルト状態を体現） |
| Insert | `#2e1a1a`（赤寄りの暗色） |
| Visual | `#2a1a3e`（紫） |

### 8.2 アニメーション一覧

| アニメーション | 秒数 | イージング |
|--------------|------|-----------|
| カード合成 | 0.3〜0.5秒 | ease-in-out |
| ドットリピートゴースト再生 | 0.15秒 | ease-out |
| undoカード分解（合成の逆再生） | 0.2秒 | ease-out |
| 画面遷移フェード | 0.2秒 | ease-out |
| ☆ロック演出 | 0.3秒 | ease-out |
| Space切替フラッシュ | 0.2秒 | ease-out |
| リザルト演出全体 | 2.5秒 | — |
| ステージイントロ | 1.5秒 | ease-out |
| 🎯フォーカスパン | 0.3秒 | ease-out |
| 不正コマンド赤フラッシュ（vignette） | 0.15秒 | ease-out |
| トースト/通知表示 | 2.0秒 | ease-out |
| lockedノードシェイク | 0.3秒（左右2往復） | ease-out |

### 8.3 キーボード入力判定

- `event.key`（論理キー）で判定。`event.code`（物理キー位置）は使用しない
- OSのキーマップに従う。Dvorak/Colemak等でも`h`を押せば`h`が入力される
- 特別な対応・注記は不要

### 8.4 テキストオブジェクト境界ルール

Vim準拠の3文字クラス分類:

| 文字クラス | 例 | 説明 |
|-----------|-----|------|
| keyword | `a-z A-Z 0-9 _` | 単語を構成 |
| punctuation | `- . , ; : ! @` 等 | 非keyword非空白。独立した「単語」 |
| whitespace | スペース、タブ | 区切り |

```
hello-world
^^^^^         keyword word → iw on 'h' = "hello"
     ^        punctuation word → iw on '-' = "-"
      ^^^^^   keyword word → iw on 'w' = "world"

aw on 'h' = "hello-"（後続の非keyword含む）
```

WORD移動（`W`/`B`/`E`）は空白のみで区切る（`hello-world` = 1つのWORD）。

### 8.5 ドットリピート（`.`）の対象ルール

| 操作 | `.`で繰り返せるか |
|------|------------------|
| 編集コマンド（`dw`, `ciw`+text, `x`, `r`, `s`+text, `>>`） | ✅ |
| Insert開始+入力（`i`+text, `o`+text, `A`+text） | ✅（入力テキスト含めて全体） |
| 移動のみ（`w`, `j`, `gg`） | ❌ |
| undo/redo（`u`, `Ctrl+R`） | ❌ |
| 検索（`/pattern`, `n`, `N`） | ❌ |
| Visualモード開始（`v`, `V`） | ❌ |
| `.`自体 | ❌（自己参照しない） |

直前の変更操作がない状態で`.`を押した場合: 何もしない。ダメージなし。

### 8.6 レジスタのステージ間持ち越し

- ステージ開始時に**全レジスタクリア**（unnamed, `"0`, `"a`〜`"z` すべて）
- ステージは独立したパズル。前ステージのヤンク内容を持ち込むと設計が破綻する

### 8.7 ヒントオーバーレイ表示仕様

| 項目 | 仕様 |
|------|------|
| トリガー | ナビゲータークリック |
| 表示方式 | エディタ領域全体を`rgba(0,0,0,0.7)`で覆い、その上に初期状態のエディタを描画 |
| デモ内容 | `optimalSpell`の各コマンドを順番に自動実行 |
| コマンド間隔 | 0.8秒 |
| コマンド表示 | オーバーレイ下部に実行中のコマンド名を順次表示 |
| 閉じ方 | 任意キー / クリック / デモ完了後1秒待機で自動閉じ |
| 閉じた後 | 元のエディタ状態に復帰（変更なし） |
| 再生中の入力 | すべてブロック（閉じる操作のみ受付） |

### 8.8 範囲外コマンドのダメージ

コマンド発行 = **1ダメージ**。実際の効果に関わらず固定。

| ケース | ダメージ | 挙動 |
|--------|---------|------|
| `5l` で行末が3文字先 | 1 | カーソルは行末まで移動 |
| `j` で最終行にいる | 1 | カーソル移動なし |
| `x` で空行にいる | 1 | 何も起きない |
| 手札にないコマンド | **0** | 不正コマンド扱い（赤フラッシュ） |

### 8.10 ステージ設計ガイドライン（概要）

| ガイドライン | 基準 |
|-------------|------|
| 1ステージの差分量 | 1〜5箇所の編集 |
| ☆3解法のコマンド数 | 3〜8コマンド |
| ライフ設計 | ☆1解法 + 余裕2〜3 = ライフ総量 |
| テキスト行数 | 序盤3〜8行、中盤8〜20行、終盤20〜40行 |
| 想定プレイ時間 | 1ステージ 30秒〜3分 |
| 複数解法 | ☆3達成可能な解法が複数あってもOK |
| 新コマンド導入 | まず1箇所のみ適用。次ステージで複数箇所 |
| テキスト言語 | チュートリアル=英文、実践=コード |

詳細なガイドラインは別途 STAGE-DESIGN-GUIDE.md として作成予定。
