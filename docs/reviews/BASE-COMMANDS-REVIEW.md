# BASE-COMMANDS-REVIEW — 基礎コマンドシステム再設計

> **ステータス**: ✅ 確定
> **作成日**: 2026-05-24
> **確定日**: 2026-05-24
> **影響範囲**: STAGE-SPEC, TUTORIAL-SPEC, ENGINE-SPEC, UI-SPEC, IMPL-SPEC

---

## 0. 背景と動機

### 0.1 現状の問題

1. **手札の視覚的ノイズ**: 後半ステージでは `h j k l w b e x i a f t 0 $` 等の基礎コマンドが手札を埋め尽くし、本来注目すべき新コマンドが目立たない
2. **データ定義の冗長性**: 全57ステージで `h j k l` を繰り返し記述
3. **パズル設計との矛盾**: 現在の `baseCommands` は「ノードの新コマンド以外は全てbase」という粒度で、ステージ設計者がパズルのために意図的に入れたコマンドまで小さく表示してしまう

### 0.2 設計目標

- **真に基本的なコマンド**のみをbase行に表示し、それ以外はステージ設計者が完全にコントロールする
- base commandsは**グローバル定数**とし、ステージ定義から除去する
- N01ノードを「基礎訓練」として再設計し、全baseコマンドを段階的に教える

## 2. N01ノード再設計

### 2.1 概要

N01を5ステージ構成の「基礎訓練ノード」に再設計する。各ステージにステージ単位のチュートリアルを付与する。

| ステージ | タイトル | 教える内容 | チュートリアル |
|----------|----------|------------|----------------|
| N01-1 | 左右に動け | `h` `l` | あり |
| N01-2 | 上下に動け | `j` `k` | あり |
| N01-3 | 単語を飛べ | `w` `b` `e` | あり |
| N01-4 | 文字を消せ | `x` + undo(`u`) + redo(`Ctrl+R`) | あり |
| N01-5 | 文字を書け | `i` `a` + `Esc` | あり |

### 2.2 N01ステージの特殊性

| 項目 | 通常ステージ | N01ステージ |
|------|-------------|------------|
| stageType | `'teach'` / `'practice'` / `'challenge'` | **`'tutorial'`**（新タイプ） |
| ライフ/ダメージ | あり | **なし** |
| ☆評価 | あり | **なし** |
| 手札表示 | base行 + HAND行 | **HAND行のみ**（まだbase未習得） |
| スキップ | — | **チュートリアルのみスキップ可**（ステージ自体はクリア必須） |
| クリア条件 | テキスト一致 | テキスト一致（goalTextで判定） |
| Weapon Get | ノードクリア後に表示 | **なし**（チュートリアルが代替） |
| ステージ間遷移 | ツリー経由 | **連続プレイ**（NEXTボタンで次へ） |
| UI教育 | — | N01-1でcube・`:q!`・`:r`を紹介 |

### 2.3 N01チュートリアル設計

#### N01-1: 左右に動け

```
テキスト: "hello!"
カーソル: (0, 0)
ゴール: カーソルを (0, 5) に移動

チュートリアル:
1. 「lを押してみろ。右に動くぞ」 → l
2. 「もう一度lだ」 → l
3. 「今度はhで左に戻れ」 → h
4. 「よし。lで右端まで行ってみろ」 → 自由移動、ゴール到達で次へ
```

#### N01-2: 上下に動け

```
テキスト:
  "first line"
  "second line"
  "third line"
カーソル: (0, 0)
ゴール: カーソルを (2, 0) に移動

チュートリアル:
1. 「jで下に移動だ」 → j
2. 「kで上に戻れる」 → k
3. 「hjklの4方向、自由に動いてみろ」 → 自由移動、ゴール到達で次へ
```

#### N01-3: 単語を飛べ

```
テキスト: "the quick brown fox jumps"
カーソル: (0, 0)
ゴール: カーソルを (0, 20) ("jumps"の先頭) に移動

チュートリアル:
1. 「wで次の単語にジャンプだ」 → w
2. 「もう一度w」 → w
3. 「bで前の単語に戻れる」 → b
4. 「eで単語の末尾に飛ぶ」 → e
5. 「wで"jumps"まで進め」 → 自由移動、ゴール到達で次へ
```

#### N01-4: 文字を消せ

```
テキスト: "hellllo world"
カーソル: (0, 3)
ゴール: "hello world" に変更

チュートリアル:
1. 「xでカーソル上の文字を消せ」 → x
2. 「もう一度xだ」 → x
3. 「おっと、消しすぎたな。uで元に戻せる」 → u
4. 「戻しすぎた？ Ctrl+Rでやり直せるぞ」 → Ctrl+R
5. 「u と Ctrl+R はいつでも使えるセーフティネットだ。覚えておけ」 → null
```

#### N01-5: 文字を書け

```
テキスト: "hllo world"
カーソル: (0, 1)
ゴール: "hello world" に変更

チュートリアル:
1. 「iを押すとカーソルの前に文字を挿入できる」 → i
2. 「eと打ってみろ」 → (Insert中、'e'入力)
3. 「Escで元のモードに戻れ」 → Esc
4. 「今度はaだ。カーソルの後ろに挿入する」 → 説明のみ(null)
5. 「Insertに入ったらEscで必ず戻る。これがVimの基本だ」 → null
```

> **注**: N01-5のチュートリアルはInsertモード内のキー入力を誘導する必要があり、チュートリアルエンジン側の対応が必要。この設計は別途検討（Q2参照）。

## 4. データモデル変更

### 4.1 グローバル定数の追加

```typescript
// engine/constants.ts（新規）
/** Base commands — always available after N01 completion, shown in compact "base" row */
export const BASE_COMMANDS = ['h', 'j', 'k', 'l', 'w', 'b', 'e', 'x', 'i', 'a'] as const

/** Commands that bypass hand restriction entirely (never shown in UI) */
export const ALWAYS_ALLOWED = ['u', 'Ctrl+R', 'Esc'] as const
```

### 4.2 Stage型の変更

```typescript
// types/stage.ts
export interface Stage {
  // ... 既存フィールド ...
  availableCommands: string[]      // ステージ固有コマンド（大きいカードで表示）
  // baseCommands は削除 → グローバル定数 BASE_COMMANDS を使用
  visualCommands?: string[]        // Visual専用（変更なし）
  // ... 既存フィールド ...
}
```

### 4.3 ステージデータからの baseコマンド除去

全ステージの `availableCommands` から `BASE_COMMANDS` に含まれるコマンドを除去する。
ただし、N01の5ステージは例外（base習得前のため、`availableCommands` にbaseコマンドを含める）。

例:
```typescript
// 旧: N18-T
availableCommands: ['h', 'j', 'k', 'l', 'w', 'b', 'e', 'x', 'dw', 'de', 'db']

// 新: N18-T
availableCommands: ['dw', 'de', 'db']
// → h,j,k,l,w,b,e,x は BASE_COMMANDS としてbase行に自動表示
```

## 5. エンジン変更

### 5.1 CommandParser

```typescript
// 手札判定の変更
getEffectiveHand(): string[] {
  const base = BASE_COMMANDS  // グローバル定数
  const hand = this.availableCommands ?? []
  const visual = (this.editorMode === 'visual' && this.visualCommands) ?? []
  return [...base, ...hand, ...visual]
}
```

### 5.2 判定フロー

```
キー入力
  → ALWAYS_ALLOWED (u/Ctrl+R/Esc) → 常に許可
  → BASE_COMMANDS → N01クリア後は常に許可
  → availableCommands → ステージ定義で許可
  → visualCommands → Visualモード中のみ許可
  → いずれにも該当しない → 無効（赤フラッシュ）
```

> **Q1: N01クリア前のbase判定** → **確定: B（availableCommandsで対応）**
> N01ステージは `availableCommands` にbaseコマンドを含める（エンジン変更不要）。N01以降のステージではbase行が自動的に表示・有効化される。N01の完了判定はUI側（base行の表示/非表示）でのみ必要。
