# チュートリアルのスコア計算 設計レビュー

## 1. 現状の問題点

### 1-1. ユーザ体験上の問題

| #   | 問題                                                             | 原因                                                                                                                                                |
| --- | ---------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| P1  | チュートリアル時はスコアが出ないことが説明されず困惑する         | チュートリアル→通常プレイの2段階構造が暗黙的                                                                                                        |
| P2  | ヒント通りに行動しても星3が得られない                            | ヒントダメージ > opt だった（A1修正で解消済み）                                                                                                     |
| P3  | リザルト画面で星ゼロなのにツリー画面で星1になっている            | `fromTutorial` 時の保存ロジック不整合                                                                                                               |
| P4  | チュートリアルが多く、スコアを得る体験になかなか到達しない       | 42/59ステージ(71%)がチュートリアル付き                                                                                                              |
| P5  | リザルト画面の"Your Spell"がプレイ途中からしか表示されない       | チュートリアル中の操作は spells に記録されない                                                                                                      |
| P6  | 全ステージ星3にはチュートリアルステージを再プレイ必要            | チュートリアル分のダメージがスコアに影響                                                                                                            |
| P7  | fromTutorial時にヒントデモを見ても実行不可能な最適解が表示される | ヒントは常に `stage.initialText` からリプレイするが、プレイヤーはチュートリアル完了後の途中状態から開始しているため、表示された操作列を再現できない |

### 1-2. 技術的な不整合

| #   | 問題                                                                                           | 箇所                                                                                  |
| --- | ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| T1  | `computeProgressUpdate` が ResultScreen で使われていない                                       | ResultScreen がインライン保存ロジックを持ち、`playMode` を無視                        |
| T2  | `LocationState` に `playMode` フィールドがない                                                 | PlayScreen が state に入れても ResultScreen で読めない                                |
| T3  | N01の `type:'tutorial'` は `life:999, stars:[999,999,999]` で実質スコア無意味だが星3として記録 | balanceValidator でも tutorial 型は公式対象外                                         |
| T4  | HintOverlay は常に `stage.initialText` / `stage.initialCursor` からデモ再生する                | `fromTutorial` で途中状態から始まっている場合、ヒントの操作列とプレイヤーの状態が乖離 |

## 2. 現状のアーキテクチャ

### 2-1. ステージ分類

```
ステージタイプ (59ステージ)
├── tutorial (10) ← N01のみ。life=999, stars=[999,999,999]
├── teach    (32) ← 全ノードの-Tステージ。全てチュートリアル付き
├── practice (13) ← -Pステージ。N13-Pのみチュートリアル付き
└── challenge (5) ← -Cステージ。チュートリアルなし
```

### 2-2. 現在のフロー

```
[チュートリアル付きステージ]
  ツリー → PlayScreen → (未完了なら) /tutorial/{id}
    → TutorialScreen: ガイド付きステップ実行（ダメージ概念なし）
    → 完了 → /play/{id} に playMode:'fromTutorial' + editorState で遷移
    → PlayScreen: damage=0 から通常プレイ開始
    → クリア → ResultScreen: 通常通りスコア計算・保存

[チュートリアルなしステージ]
  ツリー → PlayScreen → 通常プレイ → ResultScreen
```

### 2-3. 問題の構造

チュートリアル→通常プレイの2段階構造により：

- チュートリアル中のダメージはカウントされない（→P5: Spellが途中から）
- しかし通常プレイでは途中の状態から始まる（→P6: 星3にはやり直し必要）
- ユーザにはこの2段階が見えない（→P1: 困惑）

## 3. 検討した案

### 案A: チュートリアルのダメージも計算に含める

**概要**: チュートリアル中の操作も damage に加算。チュートリアルが最適解になるようステージ設計。

| メリット            | デメリット                                     |
| ------------------- | ---------------------------------------------- |
| シンプルな1段階構造 | ステージ設計が極めて困難                       |
| スコアが一貫        | f/t の違い教示など、教育目的で非最適操作が必要 |
|                     | 既存42チュートリアルの大規模リワーク           |

**判定**: ❌ 教育と最適化の両立が非現実的

### 案B: チュートリアルとステージの完全分離

**概要**: チュートリアルにはスコア概念なし。ステージ（practice/challenge）でのみスコアを得る。

| メリット               | デメリット                        |
| ---------------------- | --------------------------------- |
| スコアの一貫性         | N01であまりにもやることが多くなる |
| 教育とスコアの関心分離 | 既存のteachステージ全廃が必要     |

**判定**: △ N01の肥大化が問題

### 案C: チュートリアル付きステージのスコア廃止（★有力案）

**概要**:

- チュートリアルが存在するステージには**スコア概念をなくす**
- life=999（死なない）、ツリー画面に星表示なし
- 通常プレイ（ゼロから）もできなくし、チュートリアル→シームレスに通常プレイのフローのみ
- N01は卒業試験（N01-C）のみスコア付き
- N02以降は teach ステージを「チュートリアル専用ステージ」化し、**スコア付き練習ステージを新設**

## 4. 案C 詳細設計

### 4-1. ステージID命名規則

```
{nodeId}-{type}{suffix}

type: T=teach, P=practice, C=challenge
suffix: 省略=1つ目, a=2つ目, b=3つ目...

例:
  N13-T   ← 1つ目の teach
  N13-Ta  ← 2つ目の teach（旧N13-P）
  N27-T   ← 1つ目の teach（旧N27-1）
  N27-Ta  ← 2つ目（旧N27-2）
  N27-Tb  ← 3つ目（旧N27-3）
  N10-Pa  ← 2つ目の practice を追加する場合
```

- N01のみ例外: 現状の `N01-1`〜`N01-9`, `N01-C` を維持
- 間への挿入: `T` と `Ta` の間に `Taa` 等で対応可能
- タイプがIDから読み取れる

### 4-2. ステージ再分類

```
変更前                          変更後
──────                          ──────
N01:                            N01:
  N01-1〜9 (tutorial)             N01-1〜9 (tutorial) ← 変更なし
  N01-C    (tutorial)             N01-C    (challenge) ← 星つき卒業試験に変更

N02〜N09 (1ステージのみ):       N02〜N09:
  N0x-T (teach)                   N0x-T (teach) ← スコアなし教育専用に
                                  ★新設: N0x-P or N0x-C ← スコア付き

N10以降 (T/P/C構成):           N10以降:
  N1x-T (teach)                   N1x-T (teach) ← スコアなし教育専用に
  N1x-P (practice)                N1x-P (practice) ← 変更なし（スコア付き）
  N1x-C (challenge)               N1x-C (challenge) ← 変更なし（スコア付き）

特殊ケース:
  N13-P → N13-Ta に変更            N13-Ta (teach) ← 新コマンドを教えているため
  N27-1/2/3 → N27-T/Ta/Tb         命名規則統一
```

### 4-2. ステージタイプの再定義

| タイプ      | スコア   | 星表示   | life | チュートリアル | フロー                    |
| ----------- | -------- | -------- | ---- | -------------- | ------------------------- |
| `tutorial`  | なし     | なし     | 999  | あり           | チュートリアル→確認プレイ |
| `teach`     | なし     | なし     | 999  | あり           | チュートリアル→確認プレイ |
| `practice`  | **あり** | **あり** | 通常 | なし           | 通常プレイのみ            |
| `challenge` | **あり** | **あり** | 通常 | なし           | 通常プレイのみ            |

ポイント: **`tutorial` と `teach` を統合**して「スコアなし教育ステージ」に。

### 4-4. teach ステージのフロー（決定済み）

```
[ツリーから選択（初回・2回目以降とも）]
  → TutorialScreen（ガイド付きステップ実行）
  → PlayScreen（チュートリアル終了時の途中状態から、life=999、スコアなし）
  → クリア → 簡易リザルト

[簡易リザルトの「もう一度」]
  → PlayScreen（確認プレイからリトライ、チュートリアルスキップ）
```

- 確認プレイは必須: 教わった操作を自分で試す機会を担保
- 途中状態から継続: 最初からやり直すと「同じことを2回」のストレス
- life=999: 死なない。自由に試行錯誤できる
- ツリーから選んだら常にチュートリアルから（復習用途を想定）
- ヒントデモ: スコアに影響しないため、initialText からの再生でも問題なし（参考情報として機能）

### 4-5. 簡易リザルト画面（決定済み）

```
教育ステージ (tutorial/teach) の簡易リザルト:
┌──────────────────────────┐
│     ★ ステージクリア ★      │
│                            │
│  「不要な単語を dw で消せ」   │  ← 既存 flavor テキスト
│                            │
│  [次のステージへ]  [もう一度] │
└──────────────────────────┘
```

- **スコア/星/Your Spell**: 表示しない
- **flavor テキスト**: 既存のものをそのまま表示（将来クリア専用テキスト追加を検討）
- **「もう一度」**: 確認プレイからリトライ（チュートリアルスキップ）
- **「次のステージへ」**: 同一ノード内の次ステージへ遷移。最後のステージならツリーに戻る

### 4-7. N01-C 卒業試験（決定済み）

- N01-C を `type: 'challenge'` に変更（星・スコアあり）
- **チュートリアル（台詞のみ）は残す**: 全3ステップが台詞のみ（expectedKey: null）で操作なし
  → チュートリアルを経てもエディタ状態は initialText のまま。スコア計算に影響なし
- N01-C はゲーム内で**初めてスコアが出るステージ**
- 初回クリア時にリザルト画面の見方を解説するオーバーレイ
  - 星、ダメージ、Your Spell の意味を説明
  - 「:h でヒントも見られるが、星は1つになる」等
- stars/life の具体値は実装段階で設計する（TODO）

### 4-8. ツリー画面の変更（決定済み）

```
変更前:                     変更後:

│ N10 │                    │ N10 │
```

- `getNodeStars()` を変更: `teach/tutorial` を除外して平均
- teach のクリア状態は別のインジケータ（チェックマーク等）で表示

### 4-6. N02〜N09 のステージ新設が必要

現状これらのノードは teach ステージ1つのみ。案Cでは teach がスコア対象外になるため、

## 6. 実装注意事項

### チュートリアルスキップ時の editorState（バグ修正）

現状、スキップ時に `editorState` が渡されず `initialText` からプレイ開始になる。
修正: スキップ時もチュートリアルの全 `expectedKeys` を適用した最終 `editorState` を事前計算し、
`onComplete('skipped', finalEditorState)` で渡す。
