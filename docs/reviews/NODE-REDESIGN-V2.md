# ノード再設計 v2

> 参考記事: https://zenn.dev/hiraoku/books/vim-ni-hamaru

---

## 決定事項

### A. N01分割 → 決定

**N01: モーション基礎**（motionの概念 + 基本motion）

- hjkl（文字単位）→ w/b/e（単語単位）→ 0/^/$（行単位）→ gg/G（ファイル単位）→ f/t/;/,（行内ジャンプ）
- 「これらは緑色のカード = motion」という概念フレームを確立
- W/B/E, F/T は含めない（発展モーションへ）

**N02: 編集基礎**（base commandの残り）

- x/X（+ u/Ctrl+rもここで）→ r → i/a/I/A/o/O → .
- 「壊す → 直す → 書く → 繰り返す」の流れ
- N01+N02 = base commands

**発展モーション（構造ジャンプ）**（Visual基礎クリア後に解放）

- %, (, ), {, }, [[,]]
- テキスト構造による移動

**発展モーション（その他）**（構造ジャンプクリア後に解放）

- W/B/E, F/T, H/M/L
- 基本モーションの残り

### B. dd/cc/yyの配置 → 決定: 行単位ノードとして独立

- dd/cc/yy は operator+motion フレームワークから外れる独立した文法規則
- 「ショートカット」ではなく **Vim文法の第3の柱: オペレータ二重化 = 行単位** として教える
- dd/cc/yy に集中。>>/<<, guu/gUU は発展オペレータノードで「同じルール」と軽く触れるだけで十分
- ショートカットノード（D/C/Y/S/s/J）とは分離

### E. チュートリアルスキップ → 決定: Practice+Challengeで解放

- Tutorialはクリアでチェックマーク（任意、星なし）
- Practice + Challenge クリアでノード解放（必須）
- 既知ユーザーはTutorialを飛ばしてPracticeから開始可能
- 現行のチュートリアル=チェックマーク仕様はそのまま維持

### D. N03の役割変更 → 決定: モード概念教育ノード

- N03は「Vimの5モード思想」を教える概念ノード
- Normal/Insert: N01/N02で既習、軽く復習
- Visual: v のみ紹介（詳細はVisualノードで）
- Command-line: 基本コマンド1つだけ
- Replace: R を紹介（Replaceモードの背景色UIは後日設計）
- 「同じキーでもモードによって意味が変わる」というVim思想が主眼
- I/A/o/O は N02（編集基礎）に含める（base command）
- gi, `^ は大きい移動を学んだ後に配置（Ctrl+bで関数名を見に行った後に戻る、という体験）

### G. /,?のmotion利用 → 決定: operatorとの組み合わせは教えない

- /,? は検索専用コマンドとして教える
- operator+/,? は教えない（実用でもほぼ使わない、ゲームでは強すぎる）
- 検索+編集の実用パターンは cgn+. でカバー（Visual応用ノード）
- 提案ノード表の N10（モーション応用: op+/,?）は削除

### L. Y=y$ → 決定: neovim基準に変更

- Y = y$（カーソルから行末までヤンク）
- D=d$, C=c$ と一貫性があり、ショートカットノードで「D/C/Y = 行末まで操作」と統一的に教えられる

### M. Exコマンドの範囲 → 決定: :s + :dのみ、実装は後回し

- `:s`（置換）+ `:d`（行削除）の2つのみ
- 実装コスト: 中程度（パーサーに`:`モード追加 + 範囲パーサーが主な作業、行操作プリミティブは既存）
- `:d` は `:s` の範囲パーサーができればほぼタダ
- 他のExコマンド（`:m`, `:sort`等）は「他にも色々ある」と言及するだけ
- 実装優先度: 低（他ノード完成後でOK）

### F. ノード間の前提関係 → 決定: 依存グラフ確定

- N01→N02→N03→N04 は線形（検索まで一本道。意図的なゲーティング）
- N04 から3分岐: N05（メインルート）、N15（数値操作）、N16（スクロール+マーク）→N17（画面操作）
- N05→N06→N07→N08 は線形（operator→TextObj→行単位→Visual基礎）
- N08（Visual基礎）クリアで N09-N14 の6ノードが一斉解放
- N12（構造ジャンプ）と N14（その他）は並列（両方N08依存、順序なし）
- 発展系（N12-N14）とショートカット（N11）はおまけ的位置づけ
- N15/N16 のN04依存は内容的理由ではなくゲーティング目的

### N. Vim正規表現 → 決定: JS RegExpで代用（Vim正規表現は実装コスト的に断念）

- `:s` 実装時にJSのRegExpをそのまま使用（エスケープ行を削除するだけ）
- Vim正規表現との主な差異: `\+`→`+`, `\|`→`|`, `\(\)`→`()` が逆（変換レイヤーは30行程度だが一旦見送り）
- チュートリアル/ステージではリテラルパターンのみ出題、正規表現は教えない
- 有識者がJS正規表現の構文で書けば動く
- Vim固有の `\zs`/`\ze`/`\v`/`\V` は非対応
