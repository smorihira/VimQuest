## 変更計画

### 1. N28 y オペレータ → オペレータ系列の先頭（N07）に移動

**理由**: ヤンク（コピー）を最初に教え、その後 d（切り取り）、c（変更）の順で教えるのが教育的に最適。「d で削除したテキストも p で貼れる」という説明が自然にできる。

| 変更前                         | 変更後                            |
| ------------------------------ | --------------------------------- |
| N28: ヤンク (前提: N27 Visual) | **N07: ヤンク (前提: N01)**       |
| N10: d オペレータ (前提: N01)  | **N08: d オペレータ (前提: N07)** |
| ツリー末端でやっと学べる       | オペレータ系列の先頭で早期に学ぶ  |

- yw, ye, yb, yy, y$, y0, p, P を教える
- ステージ構成例: T1(y/yy/p) → T2(yw/ye/y$) → P → C

### 2. N10 + N11 + N12 → 統合「d オペレータ」ノード（N08）

**理由**: dw/de/db → dd → d$/d0 は全て `d` オペレータの派生であり、一つの流れで学ぶべき。

| 変更前          | 変更後                 |
| --------------- | ---------------------- |
| N10: dw, de, db | **N08: d オペレータ**  |
| N11: dd         | dw, de, db, dd, d$, d0 |
| N12: d$, d0     | （N11, N12 は廃止）    |

- 前提: N07（ヤンク。y を学んでから d を学ぶ）
- 旧 N11, N12 のステージは N08 に吸収
- ステージ構成例: T1(dw/de/db) → T2(dd) → T3(d$/d0) → P → C

### 3. 新規「c オペレータ」ノード（N09）

**理由**: d の対になる c 系コマンドを独立ノードで扱う。

| 新ノード              | コマンド               |
| --------------------- | ---------------------- |
| **N09: c オペレータ** | cw, ce, cb, cc, c$, c0 |

- 前提: N08（d を先に学んでから c を学ぶ）
- ステージ構成例: T1(cw/ce) → T2(cc) → T3(c$) → P → C
- ※ `cw` は Vim では実質 `ce` と同じ挙動（歴史的な理由）。これは teach のフレーバーテキストで触れるとよい。

### 4. N13 ショートカット → N08 の直後に配置（N10）

**理由**: D=d$, C=c$, S=cc, Y=yy のショートカット。オペレータを学んだ直後が最適。

| 変更前                         | 変更後                   |
| ------------------------------ | ------------------------ |
| N13: 前提 = N12, N22           | **N10: 前提 = N08**      |
| D, C, S のみ                   | **D, C, S, Y, J** に拡張 |
| （TextObj 習得後にやっと解禁） | （d 習得直後に解禁）     |

- D は d$ のショートカット、C は c$ のショートカット、S は cc のショートカット
- **Y は yy のショートカット**（Rev.2 で追加）
- J（行結合）も吸収（旧 N14）

### 5. N17 + N18 → 統合「オペレータ+f/t」ノード（N14）

**理由**: df/dt と cf/ct は同じ概念の d 版と c 版であり、分ける必要がない。

| 変更前      | 変更後                       |
| ----------- | ---------------------------- |
| N17: df, dt | **N14: オペレータ+f/t**      |
| N18: cf, ct | df, dt, cf, ct を 1 ノードに |

- 前提: N12（f/t）, N08（d）
- ステージ構成例: T1(df/dt) → T2(cf/ct) → P → C

### 6. TextObj 再設計（N16, N17, N18）

**現状の問題**: TextObj の概念（iw, aw, i", a" 等）と、それにオペレータを組み合わせる操作（diw, ciw 等）が4ノードに散らばっていて体系的でない。

**方針**: TextObj は2ノード（単語系・デリミタ系）に分け、オペレータとの組み合わせは1ノードにまとめる。

| 変更前                           | 変更後                                 |
| -------------------------------- | -------------------------------------- |
| N19: iw, aw                      | **N16: 単語TextObj** (iw, aw)          |
| N20: i", a", i', a', i(, a(, ... | **N17: デリミタTextObj** (i", a", etc) |
| N21: diw, di", da(               | **N18: オペレータ+TextObj**            |
| N22: ciw, ci", ca(               | diw, daw, ciw, caw, di", ci", da(, ca( |

- N16 前提: N14（オペレータ+f/t）
- N17 前提: N16
- N18 前提: N17, N08（d オペレータ）

### 7. N02 + N03 → 統合「Insert バリエーション」ノード

**理由**: I, A, o, O はすべて Insert モードに入る方法のバリエーション。基礎の i, a を学んだ直後にまとめて覚えるのが効率的。

| 変更前    | 変更後                         |
| --------- | ------------------------------ |
| N02: I, A | **N02: Insert バリエーション** |
| N03: o, O | I, A, o, O を 1 ノードに       |

- ステージ構成例: T1(I/A) → T2(o/O) → P → C

## 設計懸念（すべて解決済み）

- **N04(r) と N05(~)** → 現状維持。~ は gu/gU と統合して N19 へ移動済み
- **N06 + N07** → 現状維持 → N04(スクロール) と N05(画面位置) に再編済み
- **N08 + N09** → 統合済み → N06(検索: /, ?, n, N, \*, #)
- **N14(J)** → N10(行操作ショートカット: D, C, S, Y, J) に吸収済み
- **N24(ドットリピート)** → N11 にリナンバー、前提を N10, N02 に変更済み
- **N25(gu/gU) + N05(~)** → 統合済み → N19(大小文字操作: ~, gu, gU)
- **N28(ヤンク)** → N07 に移動。オペレータ系列の先頭（y→d→c）
