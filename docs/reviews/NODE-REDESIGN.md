## 変更計画

### 1. N10 + N11 + N12 → 統合「d オペレータ」ノード

**理由**: dw/de/db → dd → d$/d0 は全て `d` オペレータの派生であり、一つの流れで学ぶべき。

| 変更前          | 変更後                 |
| --------------- | ---------------------- |
| N10: dw, de, db | **N10: d オペレータ**  |
| N11: dd         | dw, de, db, dd, d$, d0 |
| N12: d$, d0     | （N11, N12 は廃止）    |

- 旧 N11, N12 のステージは N10 に吸収（teach ステージを統合 or 段階的 teach に再構成）
- ステージ構成例: T1(dw/de/db) → T2(dd) → T3(d$/d0) → P → C

### 2. 新規「c オペレータ」ノード

**理由**: d の対になる c 系コマンドを独立ノードで扱う。

| 新ノード              | コマンド               |
| --------------------- | ---------------------- |
| **N11: c オペレータ** | cw, ce, cb, cc, c$, c0 |

- 前提: N10（d を先に学んでから c を学ぶ）
- ステージ構成例: T1(cw/ce) → T2(cc) → T3(c$) → P → C
- ※ `cw` は Vim では実質 `ce` と同じ挙動（歴史的な理由）。これは teach のフレーバーテキストで触れるとよい。

### 3. N13 ショートカット → N10, N11 の直後に配置

**理由**: D=d$, C=c$, S=cc のショートカット。d/c を学んだ直後が最適。

| 変更前                         | 変更後                   |
| ------------------------------ | ------------------------ |
| N13: 前提 = N12, N22           | **N12: 前提 = N10, N11** |
| （TextObj 習得後にやっと解禁） | （d/c 習得直後に解禁）   |

- D は d$ のショートカット、C は c$ のショートカット、S は cc のショートカット
- d/c オペレータをやった直後に学ぶのが自然
- ノードID: **N12**（旧 N12 が廃止されるため再利用）

### 4. N17 + N18 → 統合「オペレータ+f/t」ノード

**理由**: df/dt と cf/ct は同じ概念の d 版と c 版であり、分ける必要がない。

| 変更前      | 変更後                       |
| ----------- | ---------------------------- |
| N17: df, dt | **N17: operator+f/t**        |
| N18: cf, ct | df, dt, cf, ct を 1 ノードに |

- 前提: N15（f/t）, N10（d）, N11（c）
- ステージ構成例: T1(df/dt) → T2(cf/ct) → P → C

### 5. TextObj 再設計

**現状の問題**: TextObj の概念（iw, aw, i", a" 等）と、それにオペレータを組み合わせる操作（diw, ciw 等）が4ノードに散らばっていて体系的でない。

**方針**: TextObj は一つの概念として 1〜2 ノードにまとめる。オペレータとの組み合わせ（diw, ciw 等）はノード統合済みの N10, N11 でカバーされるべき部分もあるが、TextObj 特有の組み合わせは発展ノードとして残す。

| 変更前                           | 変更後                                 |
| -------------------------------- | -------------------------------------- |
| N19: iw, aw                      | **N19: TextObj**                       |
| N20: i", a", i', a', i(, a(, ... | iw, aw, i", a", i(, a(, i{, a{ 等      |
| N21: diw, di", da(               | 全 TextObj を 1 ノードで教える         |
| N22: ciw, ci", ca(               | **N20: operator+TextObj**              |
|                                  | diw, daw, ciw, caw, di", ci", da(, ca( |

**TextObj（N19）**:

- TextObj 自体の概念を教える（inside vs around）
- 単語系: iw, aw
- デリミタ系: i", a", i(, a(, i{, a{, i[, a[
- タグ系: it, at
- ステージ構成例: T1(iw/aw) → T2(i"/a" + i(/a() → P → C
- ボリュームが大きい場合は N19（単語TextObj: iw, aw）と N19b（デリミタTextObj: i", a", i(, a( 等）に分割

**operator+TextObj（N20）**:

- d/c と TextObj の組み合わせ
- diw, daw, ciw, caw, di", ci", da(, ca(
- 前提: N19（TextObj）, N10（d）, N11（c）
- ステージ構成例: T1(diw/ciw) → T2(di"/ci" + da(/ca() → P → C

### 6. N02 + N03 → 統合「Insert バリエーション」ノード

**理由**: I, A, o, O はすべて Insert モードに入る方法のバリエーション。基礎の i, a を学んだ直後にまとめて覚えるのが効率的。

| 変更前    | 変更後                         |
| --------- | ------------------------------ |
| N02: I, A | **N02: Insert バリエーション** |
| N03: o, O | I, A, o, O を 1 ノードに       |

- ステージ構成例: T1(I/A) → T2(o/O) → P → C

## 設計懸念（すべて解決済み）

- **N04(r) と N05(~)** → 現状維持。~ は gu/gU と統合して N18 へ移動済み
- **N06 + N07** → 現状維持 → N04(スクロール) と N05(画面位置) に再編済み
- **N08 + N09** → 統合済み → N06(検索: /, n, N, \*, #)
- **N14(J)** → N09(行操作ショートカット: D, C, S, J) に吸収済み
- **N24(ドットリピート)** → N10 にリナンバー、前提を N09 に変更済み
- **N25(gu/gU) + N05(~)** → 統合済み → N18(大小文字操作: ~, gu, gU)

## 追加コマンド

### N01 で教える（既存ステージの中で）

`u`（Undo）、`Ctrl+r`（Redo）は N01 の削除（x）と一緒に教える。
