# TUTORIAL-REVIEW — インタラクティブチュートリアル設計レビュー

> **目的**: 武器取得時のステップバイステップチュートリアルの設計を議論・確定する  
> **形式**: 各項目について議論→決定の順で記録  
> **関連**: STAGE-SPEC.md（ステージ設計）、UI-SPEC.md（UI仕様）、ENGINE-SPEC.md（エンジン仕様）

---

## 背景

**現行設計**:

- 武器取得画面（Weapon Get）: 新コマンドの名前とキーを表示するだけ
- Teachステージ: 新コマンド1回で解けるステージ（変更1箇所、余裕多め）
- ヒント: ☆1コストでオーバーレイデモ

**提案（ユーザー要望）**:

- 武器取得タイミングで**スキップ可能なステップバイステップチュートリアル**を追加
- ナビゲーターが「次は`l`を押してみよう！」とつぶやく
- 指示されたキー以外は受け付けない（強制誘導）
- わざと間違えさせてundoを教える等の教育的演出
- 既存のTeachステージは変更なし（チュートリアル後にTeachに進む）

## T1: チュートリアルの発動タイミングと画面フロー

**決定**: 案A — Weapon Get直後

```
ノードクリア → [Weapon Get画面] → [チュートリアル] → ツリー画面 → [Teachステージ]
```

- 武器取得の直後に「試し振り」として発動
- チュートリアル完了（またはスキップ）後にツリー画面へ戻る
- Teachステージは変更なし（チュートリアル = 練習場、Teach = 初の実戦）

---

## T2: ステップの設計構造（データモデル）

**決定**: 1ステップ = message + expectedKey + acceptedKeys + editorSetup + wrongKeyMessage

| 要素              | 説明                                                      |
| ----------------- | --------------------------------------------------------- |
| `message`         | ナビゲーターのセリフ                                      |
| `expectedKey`     | 正解キー（nullなら任意キーで次へ）                        |
| `acceptedKeys`    | 受け付ける全キー（省略時=expectedKeyのみ）                |
| `editorSetup`     | ステップ開始時のエディタ状態（省略時=前ステップ引き継ぎ） |
| `wrongKeyMessage` | 誤入力時ナビメッセージ（省略時=デフォルト）               |

- 意図的失敗ステップ（undo教育等）もこの構造で表現可能
- 自由入力ステップ: `acceptedKeys`に複数キー + `expectedKey`に終了キー

---

## T3: ナビゲーターUIの設計

**決定**:

- キャラクター付き（最小限のアイコン+名前+セリフ）
- 表示位置: 画面下部固定バー（RPG会話ウィンドウ風）
- チュートリアル中はHAND（手札パネル）を非表示にして、そのスペースにナビを表示

---

## T4: 入力制限と誤入力の扱い

**決定**:

- 誤入力: キーは無視 + ナビが反応（期待キーを再表示）
- メッセージ: 固定（バリエーションはv2.0で検討）

---

## T5: undo/意図的失敗の教育設計

**決定**:

- undoチュートリアル: N02（x初習得時）に組み込む
- 流れ: x実行→消しすぎ→ナビ「uを押せ」→undo体験→「いつでも使える安全網」説明
- T2のステップ構造で表現可能（特別な仕組み不要）

---

## T6: スキップ機能の設計

**決定**:

- 粒度: チュートリアル全体スキップ（ステップ単位不可）
- 操作: Esc + 画面隅「Skip」ボタン（両方）

## T8: チュートリアルとTeachステージの関係

**決定**: 両方残す（2段階）

- チュートリアル = ガイド付き練習（指示通り押すだけ）
- Teach = 初の自力プレイ（ガイドなし、簡単）
- 冗長に感じる人はチュートリアルをスキップ

---

## T9: チュートリアルのデータ形式と管理

**決定**:

- 専用ディレクトリ: `data/tutorials/N01.ts` 等
- ステージデータとは分離（型が異なるため）
- TutorialStep型で定義

```typescript
interface TutorialStep {
  message: string
  expectedKey: string | null
  acceptedKeys?: string[]
  editorSetup?: { text: string; cursor: { line: number; col: number } }
  wrongKeyMessage?: string
}
```

---

## T10: 進行状態の永続化

**決定**:

- 完了/スキップのみ保存（途中状態は不要）
- GameProgressに `tutorialStatus: Record<string, 'completed' | 'skipped'>` を追加

## T12: チュートリアル再閲覧機能

### 現状の問題

1. **一度完了/スキップしたチュートリアルは二度と見られない**
   - `tutorialStatus` に記録されると `showTutorial = false` になり表示されない
   - 忘れた時に復習する手段がない
2. **TutorialScreen (`/tutorial/:nodeId`) は nodeId ベース**
   - 「ノードの最初のチュートリアル」しか表示できない（stageId指定不可）
   - N01のように複数チュートリアルを持つノードでは個別のチュートリアルにアクセスできない
3. **プレイ画面からチュートリアルにアクセスする手段がない**
   - `:h` はヒント表示（答えの自動再生）であり、チュートリアルとは別物

### 案A: プレイ画面に「📖」ボタン追加（最小変更）

チュートリアルがあるステージのプレイ画面TopBarに📖ボタンを追加。押すとチュートリアルを再表示。

```
[ :q! ] [ :e! ] [ :h ] [ 📖 ] [ 🔊 ]
```

**変更範囲**:

- `TopBar.tsx`: 📖ボタン追加（`hasTutorial(stageId)` で表示制御）
- `PlayScreen.tsx`: `showTutorial` 状態に「手動トリガー」フラグ追加
- `StageTutorial.onComplete`: 手動起動時は `tutorialStatus` を上書きしない

**メリット**: 変更量小。「今やってるステージの操作を確認したい」を直接解決。  
**デメリット**: 他のステージのチュートリアルは見られない。

### 案B: ステージ選択パネルに「📖」アイコン追加

SkillTreeのステージ選択パネルで、チュートリアルがあるステージの横に📖ボタンを表示。

```
┌──────────────────────┐
│ N01  基礎訓練      ✕ │
│ ★ 左右に動け ★★★ 📖 │
│ ★ 上下に動け ★★★ 📖 │
│ ★ 単語を飛べ ★★★ 📖 │
│ ...                   │
└──────────────────────┘
```

**変更範囲**:

- `SkillTreeScreen.tsx`: ステージ選択パネルに📖ボタン追加
- ルーティング: `/tutorial/:nodeId` → `/tutorial/:stageId` に変更
- `TutorialScreen.tsx`: stageIdからチュートリアル取得するように修正

**メリット**: 任意のステージのチュートリアルにアクセス可能。一覧性あり。  
**デメリット**: プレイ中には確認できない（ツリーに戻る必要あり）。

### 案C: A + B の併用

プレイ画面の📖（案A）＋ ツリーのステージ選択パネルの📖（案B）を両方実装。

**メリット**: プレイ中にもツリーからも復習可能。両方のユースケースをカバー。  
**デメリット**: 変更量がやや多い（ただし複雑度は低い）。

### 案D: チュートリアル一覧ページ追加

`/tutorials` ルートを新設し、全チュートリアルの一覧ページを表示。

```
┌──────────────────────────────────────┐
│ TUTORIAL ARCHIVE                      │
│                                       │
│ N01 基礎訓練                          │
│   N01-1  左右に動け (h/l)        [▶]  │
│   N01-2  上下に動け (j/k)        [▶]  │
│   ...                                 │
│                                       │
│ N15 行内ジャンプ (将来)               │
│   (coming soon)                       │
└──────────────────────────────────────┘
```

**メリット**: 全体俯瞰で復習。将来チュートリアル増加に対応。  
**デメリット**: 変更量大。現時点ではN01の9件のみで一覧ページの価値が薄い。ゲーム導線から外れる。

### 論点

1. **案の選択**: → **案C（A+B併用）に決定**
2. **ルーティング変更**: → `/tutorial/:stageId` に変更。nodeIdアクセス時はそのノードの最初のチュートリアルにフォールバック
3. **手動再閲覧時の `tutorialStatus` 管理**: → 再閲覧時は更新しない。`isReview` propsで制御
4. **キーボードショートカット**: → **不要**。一部ステージのみなのでボタンで十分
