# チュートリアル × コマンド設計表

> 各ノードのステージ構成と、チュートリアルが教えるコマンドの全体マップ。
> `newCommands` はステージセレクタに表示され、「このステージで何を学ぶか」を示す。

## 設計方針

- **ステージ順序**: 全チュートリアル（T）→ Practice（P）→ Challenge（C）
- **Practice**: 実践的に有り得そうなシチュエーション。テクニック系（ddからのpで行入替など）
- **Challenge**: パズル要素の強い頭を使うステージ（実践要素は薄くてよい）。追々追加する
- **N03以降**: 常にドットリピート（`.`）を念頭に置きながらステージを作成する
- **(復習)**: newCommands 空配列 — 既出コマンドの応用・組み合わせ

## 凡例

| 記号 | 意味                     |
| ---- | ------------------------ |
| ✅   | 実装済み                 |
| 🔲   | 未実装（要作成）         |
| 🔄   | 既存ステージの変更が必要 |
| 🗑️   | 削除対象                 |
| 🆕   | 新規ステージ             |

## 実装計画（ステージ再設計の前提タスク）

チュートリアル・ステージの再設計に着手する前に、エンジン側で必要な機能追加・修正を行う。

### Phase 0: 小規模修正（既存コードの変更のみ）

| #   | タスク           | 内容                                                                | 影響範囲                                 |
| --- | ---------------- | ------------------------------------------------------------------- | ---------------------------------------- |
| 0-1 | `Y` を y$ に修正 | 現状 yy（行ヤンク）→ Neovim 準拠の y$（カーソル〜行末ヤンク）に変更 | `commandExecutor.ts`                     |
| 0-2 | `gN` 実装        | gn の逆方向版。検索マッチを後方選択                                 | `commandParser.ts`, `commandExecutor.ts` |

### Phase 1: 新機能実装

| #   | タスク              | 内容                                                                     | 影響範囲                                                  |
| --- | ------------------- | ------------------------------------------------------------------------ | --------------------------------------------------------- |
| 1-1 | Visual Block Insert | Ctrl+v 選択後に `I` で各行先頭に挿入、`A` で各行末尾に挿入               | `commandExecutor.ts`, `insertMode.ts`                     |
| 1-2 | `:s` コマンド       | Command-line モードで `:s/old/new/[g]` による置換。`:%s` もサポート      | `commandParser.ts`, 新ファイル `commandLine.ts`（要検討） |
| 1-3 | `+` レジスタ実接続  | `"+y` で `navigator.clipboard.writeText()`、`"+p` で `readText()` を呼ぶ | `commandExecutor.ts` の register 処理                     |

### Phase 2: 基盤改修

| #   | タスク              | 内容                                                                                                                                             | 影響範囲                                             |
| --- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------- |
| 2-1 | Base コマンド段階化 | 固定の `BASE_COMMANDS` → ノード進行に応じて成長する仕組みに変更。N01-P 時点では N01 コマンドのみ、N02-P で N02 コマンド追加、N03 以降はフル base | `commandParser.ts`, `skillTree` 周り, ステージデータ |

### 実装順序

```
0-1 (Y修正) → 0-2 (gN) → 1-1 (Block Insert) → 1-2 (:s) → 1-3 (+register) → 2-1 (Base段階化)
```

- Phase 0 は単純な修正なので即着手可能
- Phase 1 は互いに独立しているが、1-1 → 1-2 の順が自然（Visual Block → Command-line）
- Phase 2 はステージデータの設計に直結するため、ステージ再作成の直前に行う
