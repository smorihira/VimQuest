## 目的

1. **全111コマンドをチュートリアルで網羅する**（現状67/111 = 60%のカバー率）

### B. グルーピングの問題

- **Ctrl+a / Ctrl+x**: 本質的に同じ操作の+/-。別チュートリアルに分離されている
- **dw / de / db**: dw と de は対比で教えるが、db は完全に省略
- **cw / ce / cb**: 同上
- **p / P**: p のみ教えて P は省略。「下/上」の対比が欠落

### C. 教え方の精度

- **dw の説明**: 「空白ごとスッキリ消える」→ カーソル位置から次の単語の先頭まで削除、の説明がない
- **de の説明**: 「単語だけ消える…が空白が残る」→ カーソル位置から単語末尾まで削除、の説明がない
- **diw との違い**: dw は「カーソル位置から」、diw は「単語全体」。この対比が不明確
- **cw の特殊挙動**: Vim では cw ≒ ce（歴史的理由）だが、本エンジンでは cw = c+w（末尾空白含む）。この差の説明がない

## 設計原則

### 原則1: 対比で教える

同じ概念の正反対/バリエーションは1つのチュートリアルで対比して見せる。

- `dw` vs `de` vs `db` → 3つを1チュートリアルで
- `Ctrl+a` vs `Ctrl+x` → 1チュートリアルで
- `p` vs `P` → 1チュートリアルで
- `i` vs `a` → 1チュートリアルで（現状もそう）
- `f` vs `t` → 1チュートリアルで（現状もそう）
- `n` vs `N` → 1チュートリアルで（現状もそう）

### 原則2: 概念を教えたらバリエーションはまとめて紹介

- `i"` を教えたら `i'`, `i(`, `i{` は「同じ要領で他のデリミタにも使える」と一言添える
- TextObj の `i` vs `a` の違いは一度だけ丁寧に教える。以降は省略可

### 原則3: 「カーソル位置から」を明示する

motion 系（w, e, b, $, 0）を使うコマンドは必ず「カーソル位置から○○まで」と説明する。
TextObj 系（iw, aw, i"）は「カーソルがどこにいても単語/デリミタ全体」と対比する。

### 原則4: チュートリアルは teach ステージのみ

practice / challenge にはチュートリアルを付けない（現状の方針を維持）。
ただし teach ステージの中で全コマンドの基本を網羅する。

## 教え方の改善: 正確な仕様説明

### motion 系オペレータの統一フォーマット

すべての motion 系コマンドの説明に「カーソル位置から」を明示する。

| コマンド | 現行の説明                | 改善案                                                   |
| -------- | ------------------------- | -------------------------------------------------------- |
| dw       | 空白ごとスッキリ消える    | **カーソル位置から次の単語先頭まで**削除（空白含む）     |
| de       | 単語だけ消える            | **カーソル位置から単語末尾まで**削除（空白は残る）       |
| db       | （なし）                  | **カーソル位置から前の単語先頭まで**削除（逆方向）       |
| d$       | （d0 の対として言及のみ） | **カーソル位置から行末まで**削除                         |
| d0       | 行頭まで一気に消える      | **カーソル位置から行頭まで**削除                         |
| dd       | 行まるごと消える          | **行全体**を削除（カーソル位置に関係なく）               |
| diw      | （N16/N18で教育）         | **カーソルが載っている単語全体**を削除（位置に関係なく） |

### dw vs diw の対比（必ず教えるべき）

> dw は「カーソルから先」を消す。単語の途中にいたら途中から消える。
> diw は「単語まるごと」を消す。単語のどこにカーソルがあっても結果は同じ。

この対比は N16-T（TextObj 初登場時）で明示する。

### cw の特殊挙動

本エンジンの cw は c+w motion（次の単語先頭まで変更 = 末尾の空白も含む）。
Vim 本来の cw は ce 相当（歴史的特例）。

→ **教え方**: 「cw はカーソル位置から次の単語先頭まで変更する」とだけ説明。
Vim との差異はフレーバーテキストで触れる程度にとどめる。

## 議論ポイント

### 議論1: N15 統合の是非

Ctrl+a と Ctrl+x を統合して1つの teach ステージで教えるべきか？

- **統合賛成**: 本質的に +/- の対。分ける意味がない
- **統合反対**: 各ステップを体験させるには分けた方が丁寧

### 議論2: N07 チュートリアルの大幅修正

現行は yiw → p の流れだが、ノードの commands は yw。

- **案A**: yiw を残しつつ yw/ye/yb も教える（ステップ数が多くなる）
- **案B**: yiw を削除して yw から教え直す（シンプル）

### 議論3: デリミタ TextObj の網羅度

i"/a" を教えれば i'/i(/i{ は推論できるが、チュートリアルで一度も触れないのは不安。

- **案A**: null ステップで一覧を見せるだけ
- **案B**: 2〜3種類を実際に操作させる

### 議論4: 新規 teach ステージの追加方針

N04-Ta, N05-Ta, N05-Tb の新設は stageCount を変更する。

- これにより balanceValidator テストにも影響
- ステージデータ + チュートリアルデータの両方が必要

### 議論5: N08 dw/de/db の説明改善の詳細

「カーソル位置から」を入れるだけで十分か？ 実際にカーソルを途中に置いて dw/diw の差を体験させるべきか？
